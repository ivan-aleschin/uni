# THEORY — Лабораторная работа №2: АЛУ (Arithmetic Logic Unit)

## 1) Что такое АЛУ и зачем оно нужно
АЛУ (Arithmetic Logic Unit) — это «калькулятор» процессора.  
На входы подаются два операнда `a_i`, `b_i` и код операции `alu_op_i`.  
На выходе появляются:
- `result_o` — 32‑битный результат вычислительной операции;
- `flag_o` — 1‑битный флаг сравнения.

### Два независимых режима
В архитектуре курса:
- вычислительные опкоды пишут результат в `result_o`, а `flag_o = 0`;
- опкоды сравнения пишут флаг в `flag_o`, а `result_o = 0`.

Это важно для дальнейших лабораторных работ, где используется сигнал `flag`.

---

## 2) Операции АЛУ

### Вычислительные (result_o)
- `ADD`  — сложение (через сумматор из ЛР1)
- `SUB`  — вычитание (оператор `-`)
- `AND`  — побитовое И
- `OR`   — побитовое ИЛИ
- `XOR`  — побитовое XOR
- `SLL`  — логический сдвиг влево
- `SRL`  — логический сдвиг вправо
- `SRA`  — арифметический сдвиг вправо (с сохранением знака)
- `SLTS` — знаковое сравнение `<` (результат 0/1)
- `SLTU` — беззнаковое сравнение `<` (результат 0/1)

### Сравнения (flag_o)
- `EQ`  — равенство
- `NE`  — неравенство
- `LTS` — знаковое `<`
- `GES` — знаковое `>=`
- `LTU` — беззнаковое `<`
- `GEU` — беззнаковое `>=`

## 3) Важные нюансы SystemVerilog

### 3.1 Комбинаторная логика
Для чистой комбинации используют `always_comb`:
- нет задержек
- нет `posedge`
- все выходы должны иметь значение при любом опкоде (т.е. нужен `default`)

### 3.2 Знаковые операции
По умолчанию все сигналы беззнаковые.  
Для знаковых операций применяют `$signed(...)`:

- `SRA`: `$signed(a_i) >>> b_i[4:0]`
- `SLTS`, `LTS`, `GES`: `$signed(a_i) < $signed(b_i)`

### 3.3 Сдвиги: только `b[4:0]`
В RISC‑подобной логике используется **только 5 младших бит** операнда `b`:
```
shift = b_i[4:0]
```
Это означает: сдвиги на 0–31.

## 6) Типичный шаблон логики (идея)
- один `always_comb` для `result_o`
- второй `always_comb` для `flag_o`
- на каждом: `case (alu_op_i)` + `default`

Так проще контролировать, чтобы `result` и `flag` не активировались одновременно.
