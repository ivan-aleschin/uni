# Лабораторная работа №6 — Процессорная система RISC-V с LSU и прерываниями

Данная работа является финальным этапом интеграции процессорного ядра. Она объединяет три ключевых модуля из репозитория APS: **ЛР7 (Тракт данных)**, **ЛР9 (LSU)** и **ЛР11 (Прерывания)**.

---

## 🏗️ Архитектура и Схема

Система построена по классической однотактной схеме RISC-V с расширениями для работы с памятью и обработки внешних событий.

### 1. Тракт данных (Datapath - ЛР7)
Это "скелет" процессора. Включает в себя:
- **Core (Ядро):** Координирует работу всех блоков.
- **Decoder:** Переводит 32 бита команды в управляющие сигналы.
- **ALU (АЛУ):** Выполняет вычисления.
- **Register File:** 32 регистра по 32 бита. Регистр `x0` всегда равен 0 (аппаратный ноль).
- **PC Logic:** Рассчитывает адрес следующей инструкции.

### 2. LSU (Load-Store Unit - ЛР9) — Глубокий разбор
LSU — это интеллектуальный интерфейс между ядром (которое хочет работать с 32-битными числами) и памятью (которая хранит данные побайтово).

#### Зачем он нужен?
Память в RISC-V побайтовая, а регистры — 32-битные. Если нам нужно прочитать только один символ (1 байт), мы не можем просто засунуть его в регистр, так как останутся лишние 24 бита. LSU решает эту задачу.

#### Инструкции загрузки (Load)
*   **lb (Load Byte):** Читает 8 бит. Чтобы число сохранило свой смысл, LSU выполняет **Sign Extension (Знакорасширение)** — копирует 7-й бит байта (знаковый) во все старшие 24 бита регистра.
*   **lbu (Load Byte Unsigned):** То же самое, но вместо знака слева дописываются **нули**. Используется для беззнаковых данных (коды символов, цвета).
*   **lh / lhu (Load Halfword):** Работает аналогично с 16 битами (2 байта).

#### Инструкции записи (Store) и Byte Enable
*   **sb (Store Byte) / sh (Store Halfword):** Позволяют изменить только часть 32-битного слова в памяти.
*   **Byte Enable (be_i):** LSU генерирует 4-битную маску (например, `4'b0001`), которая разрешает памяти записывать данные только в конкретный байт, не затирая соседние.

#### Логика Stall (Приостановка)
Мы используем **Block RAM** (блочную память ПЛИС). Её физическая особенность в том, что она выдает данные с задержкой в **1 такт**. 
*   LSU выставляет сигнал `stall_o = 1` при любом запросе к памяти.
*   Ядро «замораживает» значение PC и текущую инструкцию на один такт.
*   На следующем такте данные из памяти стабилизируются, попадают в регистр, и `stall` падает в 0.

### 3. Подсистема прерываний (Interrupts - ЛР11)
- **CSR Controller:** Содержит системные регистры (`mtvec` — адрес обработчика, `mepc` — адрес возврата).
- **Interrupt Controller:** Следит за внешним сигналом `irq_req`. Если прерывания разрешены (бит `mie` в `mstatus`), он заставляет ядро совершить прыжок на адрес обработчика.

## 📈 Как проверить работу (Временная диаграмма)

При защите в **GTKWave** или **Vivado Waveform** смотрите на эти моменты:

### 1. Обращение к памяти (Load/Store)
- Найдите инструкцию `lw` или `sw` (например, на адресе `0x30`).
- **Критический момент:** В этот такт сигнал **`core_stall`** должен стать **1**.
- Убедитесь, что **`instr_addr` (PC)** не меняется в следующем такте (процессор ждет память).
- На сигнале **`ext_mem_be`** посмотрите маску записи (для `sw` она будет `F`, для `sb` — один из битов).

### 2. Обработка прерывания (Trap)
- Когда `irq_req_i` становится 1, на следующем такте `PC` должен прыгнуть на адрес из регистра `mtvec`.
- Сигнал `trap` в ядре должен быть равен 1 в момент прыжка.

### 3. Возврат из прерывания (MRET)
- Инструкция `mret` (код `30200073`) должна вернуть `PC` к значению, которое было в `mepc`.
