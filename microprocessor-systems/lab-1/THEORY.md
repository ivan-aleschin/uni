# ЛР1 — THEORY: Сумматор и базовый SystemVerilog

## 1. О чём лабораторная
ЛР1 знакомит с описанием цифровых схем на SystemVerilog и с базовыми логическими операциями. Итог — иерархия сумматоров: `fulladder` → `fulladder4` → `fulladder32`.  
Это фундамент для АЛУ и всего процессора.

---

## 2. Ключевые термины
- **Сумматор (Adder)** — устройство, складывающее двоичные числа.
- **Полный сумматор (Full Adder)** — 1‑битный блок с входами `a`, `b`, `carry_in` и выходами `sum`, `carry_out`.
- **Иерархия модулей (Module Hierarchy)** — сборка сложного устройства из простых блоков.
- **Комбинаторная логика (Combinational Logic)** — выход зависит только от текущих входов, без памяти.

---

## 3. Базовый синтаксис SystemVerilog
- **Модуль** объявляется через `module ... endmodule`.
- **Порты** описываются как `input`, `output`, `logic`.
- **Непрерывные присваивания**: `assign out = expr;`  
  Используется для комбинаторной логики.
- **Векторные сигналы**: `logic [31:0] a;` — 32‑битный сигнал.

---

## 4. Как работает 1‑битный fulladder
Формулы:
- `sum = a XOR b XOR carry_in`
- `carry_out = (a AND b) OR (a AND carry_in) OR (b AND carry_in)`

---

## 5. Иерархия сумматоров
- `fulladder4` — цепочка из четырёх `fulladder`.
- `fulladder32` — цепочка из восьми `fulladder4`.

Перенос (carry) передаётся между блоками по цепочке — это «сложение в столбик» в цифровой логике.

---

## 6. Тестирование
**Тестбенч (Testbench)** — несинтезируемый модуль для симуляции.  
Что важно:
- тестбенч не прошивается в FPGA;
- тестбенч задаёт входы, пишет `wave.vcd` и проверяет выходы.

Проверка:
- перебор всех входов для 1‑битного сумматора;
- несколько кейсов для 32‑битного сумматора.

---

## 7. Типичные ошибки
- Неправильные имена портов при подключении.
- Перепутан `carry_in` / `carry_out`.
- Несоответствие разрядностей (например, 1‑бит к 4‑битному).
- Пропущенный файл в командной строке компиляции.

---

## 8. Vivado и ПЛИС (FPGA)
**Vivado** — САПР для синтеза и прошивки FPGA (ПЛИС).  
Минимальный цикл:
1. Создать проект RTL.
2. Добавить `design.sv` в Design Sources.
3. Добавить тестбенчи в Simulation Sources.
4. Запустить симуляцию.
5. Синтез → Implementation → Generate Bitstream.
6. Program Device (прошивка).
