# THEORY — Лабораторная №5 (Типы инструкций RISC-V)

## 1. Разбивка инструкции по форматам
RISC-V использует фиксированную длину 32 бита, но разную структуру полей:
* **R-type:** Работает только с регистрами (ADD x1, x2, x3).
* **I-type:** Регистр + маленькое число (ADDI x1, x2, 10).
* **S-type:** Запись в память (SW x1, 0(x2)).
* **B-type:** Ветвление (BEQ x1, x2, label).
* **U-type:** Большое число (LUI).

## 2. Поле Opcode [6:0]
Это "ID" команды. По первым 7 битам декодер понимает, к какому типу относится инструкция. 
* *Важно:* В RISC-V два младших бита опкода всегда `11`. Если там `00` или `10` — это нелегальная инструкция (`illegal_instr_o = 1`).

## 3. Обработка ошибок
Если декодер видит неизвестный код операции, он обязан выставить `illegal_instr_o`. Физически это блокирует запись в регистры и память, чтобы процессор не испортил данные при сбое.
