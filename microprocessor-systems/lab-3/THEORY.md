# THEORY — Лабораторная работа №3
**Тема:** регистровый файл и память инструкций  
**Цель:** понять модели памяти в цифровых схемах, научиться описывать память на SystemVerilog и корректно использовать адресацию.

---

## 1) Что мы строим в ЛР3
В этой работе два ключевых блока:

1) **Память инструкций (Instruction Memory, IMEM)**  
   - ROM (Read‑Only Memory)  
   - Асинхронное чтение  
   - Инициализируется из `program.mem` через `$readmemh`

2) **Регистровый файл (Register File, RF)**  
   - 32 регистра × 32 бита  
   - 2 асинхронных порта чтения  
   - 1 синхронный порт записи  
   - Регистр `x0` всегда равен 0

Эти блоки нужны в процессоре всегда: IMEM отдаёт следующую инструкцию, RF хранит операнды и результаты.

---

## 2) Модели памяти: асинхронная vs синхронная

### Асинхронная память
- Выход зависит **только от текущего адреса**.
- Данные появляются сразу (в пределах задержки модели).
- Удобно для IMEM — инструкция должна быть доступна сразу.

### Синхронная память
- Чтение и/или запись происходят **по фронту такта**.
- Такая модель ближе к реальной SRAM/BRAM.

**В ЛР3:**
- IMEM — асинхронное чтение.
- RF — асинхронное чтение, синхронная запись.

---

## 3) Регистровый файл: зачем 2 порта чтения
АЛУ имеет **два входа**, значит нужно уметь получить два операнда за один такт.  
Поэтому RF имеет два независимых порта чтения (`read_data1`, `read_data2`).

Запись идёт по фронту такта — это защищает от гонок и делает поведение предсказуемым.

---

## 4) Аппаратный ноль (x0)
В архитектуре RISC‑V регистр `x0` **всегда равен 0**.  
Это упрощает программы:
- `x1 = x2 + x0` → копирование
- сравнение с нулём без отдельного регистра

Типичная реализация:
- запись в `x0` запрещена  
- чтение `x0` всегда возвращает 0

---

## 5) Адресация памяти: побайтовая логика
В процессорах используется **byte addressing**: адрес — это адрес байта.  
Но память инструкций хранит **слова по 32 бита** (4 байта).

Следовательно, индекс слова = `addr / 4`.  
В двоичном виде это значит отбросить 2 младших бита:

```
index = addr[10:2]
```

Если IMEM = 512 байт, то:
- 512 / 4 = 128 слов  
- адрес слова = 7 бит  

---

## 6) `$readmemh`: как загружается программа
`$readmemh("program.mem", ROM);`

Файл `program.mem` должен содержать **HEX**:
```
00000013
00400093
...
```

- 1 строка = 1 слово (32 бита)
- если строк меньше, симулятор выдаёт предупреждение
- чтобы избежать предупреждений, можно дописать нулями до конца массива

---

## 7) SystemVerilog: ключевые конструкции

### Массив памяти
```
logic [31:0] ROM [0:WORDS-1];
```

### Асинхронное чтение
```
assign read_data_o = ROM[addr_index];
```

### Синхронная запись
```
always_ff @(posedge clk) begin
  if (we) mem[wa] <= wd;
end
```

### Комментарий по стилю
- `assign` = комбинационная логика
- `always_ff` = последовательная логика (такт)

---

## 8) Проверка в симуляции (GTKWave)

### Что смотреть
- `clk`
- `write_enable`
- `write_addr`, `write_data`
- `read_addr1`, `read_addr2`
- `read_data1`, `read_data2`

### Как должна вести себя память
- `read_data` меняется мгновенно при смене `read_addr`
- `write_data` попадает в память **только по фронту**
