# THEORY — Лабораторная №3 (Регистр x0 и Адресация)

## 1. Регистр x0 — Аппаратный ноль
В RISC-V регистр номер 0 всегда равен 0. 
* **Физическая реализация:** На портах чтения стоит мультиплексор: `assign rd = (addr == 0) ? 0 : mem[addr]`.
* **Зачем?** Это упрощает команды. Например, `addi x1, x0, 5` превращается в обычное `x1 = 5`.

## 2. Байтовая адресация vs Индексы массива
Процессор выдает **байтовый адрес** (0, 4, 8, 12...). Но массив в коде имеет индексы (0, 1, 2, 3...). 
* **Математика:** Чтобы из адреса `0x4` получить индекс `1`, нужно разделить на 4. 
* **Сдвиг:** Деление на 4 — это отбрасывание двух младших бит. Поэтому в коде мы пишем `addr[10:2]`.

## 3. Синхронность
* **Чтение:** Асинхронное. Как только вы подкрутили "ручку" адреса, данные на выходе сменились. Это экономит время.
* **Запись:** Синхронная. Нужен удар `clk`, чтобы данные "протолкнулись" в триггеры памяти.
