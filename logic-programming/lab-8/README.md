## Задание 1. `nu(X, Y)`

**Условие:** реализовать аналог `\+ (X = Y)` через `!` и `fail`.

**Как работает:**
1. Если `X` и `Y` унифицируются (клоз 1), срабатывает `!`, отсекающий клоз 2.
2. `fail` вызывает неудачу всего предиката.
3. Если `X` и `Y` не унифицируются, клоз 1 пропускается, срабатывает клоз 2 и возвращает истину.

**Описание:** Пример использования паттерна cut-fail для реализации отрицания как неудачи.

## Задание 2. `abs(X, Abs)`

**Условие:** вычислить абсолютное значение числа с использованием зеленых отсечений.

**Как работает:**
1. Если `X >= 0`, первый клоз вычисляет `Abs = X`.
2. `!` отсекает проверку альтернативы `X < 0` при бэктрекинге.
3. Если `X < 0`, срабатывает второй клоз.

**Описание:** Зеленое отсечение оптимизирует выполнение, не меняя логику программы.

## Задание 3. `set_union(L1, L2, Res)`

**Условие:** объединение двух списков как множеств (без дубликатов).

**Как работает:**
1. Для каждого элемента `H` из первого списка проверяем, есть ли он во втором (`member`).
2. Если есть — `!` отсекает добавление и рекурсия идет дальше.
3. Если нет — элемент добавляется в результат.

**Описание:** Демонстрация использования отсечения для управления ветвлением логики.

## Задание 5. `unifiable(List, Term, Res)`

**Условие:** найти элементы списка, унифицируемые с термом без связывания переменных.

**Как работает:**
1. Используем двойное отрицание `\+ \+ (H = Term)`.
2. Первая попытка унификации связывает переменные, но `\+` отменяет эти изменения.
3. Второе `\+` возвращает истину, если попытка была успешной.

**Описание:** Метод проверки возможности унификации без побочных эффектов.
