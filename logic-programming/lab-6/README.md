## Задание 1. Построение деревьев вызова (call trees) и поведение предикатов

Ниже приведены разборы и деревья вызова для двух типичных запросов, которые часто используются при отладке и анализе программ на Прологе.

1) Запрос:
?- предок_потомок(л��за, X).  %% family.pl

Предполагается, что в `family.pl` определён предикат `предок_потомок/2`, реализующий переход через `parent/2` (как предок или транзитивно через потомков). Пусть база содержит факты:
```
parent(ivan, mary).
parent(mary, lisa).
```
Тогда запрос: `предок_потомок(mary, X).` даст `X = lisa.`

Дерево вызова для `предок_потомок(lisa, X)` (по аналогии) показывает, как Пролог разворачивает правила:

- предок_потомок(л��за, X)
  - попытка первого клоза: parent(л��за, X) — (проверка фактов)
    - если факт есть — возвращаем ответ
  - если первый клоз не дал ответа, то пробуем второй (рекурсивный):
    - parent(л��за, Z), предок_потомок(Z, X)
    - для каждого Z, найденного по parent/2, делаем рекурсивный вызов предок_потомок(Z, X)
    - рекурсия продолжается, пока не будут найдены либо все варианты, либо исчерпаны факты

Пример конкретного дерева (в упрощённом виде), если в базе:
```
parent(stephen, ivan).
parent(ivan, mary).
parent(mary, lisa).
```
Запрос `?- предок_потомок(stephen, X).` разворачивается так:
- пробуем parent(stephen, X): находим X = ivan  (ответ 1)
- рекурсивно: для найденного ivan вызываем предок_потомок(ivan, X)
  - parent(ivan, X): находим X = mary  (ответ 2)
  - рекурсивно: предок_потомок(mary, X)
    - parent(mary, X): находим X = lisa  (ответ 3)
    - рекурсия завершается, когда не остаётся новых фактов

Таким образом, результатом будут все потомки stephen: ivan, mary, lisa (в порядке генерации).

2) Запрос:
?- my_last2(X, a).  %% lists.pl

Предположим, `my_last2(Item, List)` определён как "Item — последний элемент списка List". Вариант запроса `my_last2(X, a)` некорректен (второй аргумент `a` не список), поэтому Пролог вернёт `false` или ошибку типов в зависимости от реализации предиката `my_last2/2`. Ожидаем правильное поведение при использовании корректных аргументов, например:
```
?- my_last2(X, [b, c, a]).
X = a.
```

Дерево вызова для `my_last2(X, [b,c,a])` (типичная реализация через рекурсию):
- my_last2(X, [b,c,a])
  - [b|Tail] — вызываем my_last2(X, [c,a])
    - my_last2(X, [c,a])
      - [c|Tail2] — вызываем my_last2(X, [a])
        - my_last2(X, [a])  % базовый случай: список из одного элемента
          - unify X = a
        - возвращаем X = a вверх по стеку

Если в запросе аргументы перемешаны, например `?- my_last2([a,b], X).`, то предикат должен вернуть `X = b` если он реализован для списка `[a,b]`. Если аргументы находятся в других местах, поведение зависит от реализации — некоторые реализации могут работать только при конкретном направлении (list instantiated), другие — быть более общими.

## Задание 2. `zip(L1, L2, Res)`

**Условие:** объединить два списка одинаковой длины в список пар.

**Как работает:**
1. Базовый случай: два пустых списка дают пустой список.
2. Головы `H1` и `H2` объединяются в список `[H1, H2]`.
3. Рекурсия продолжается для хвостов.

**Примеры и поведение с переменными:**
- `?- zip([a,b,c], [1,2,3], X).`
  - Результат: `X = [[a,1],[b,2],[c,3]]`.
- `?- zip(X, [1,2,3], [[a,1],[b,2],[c,3]]).`
  - Результат: `X = [a,b,c]`.
- `?- zip([a,b], Y, [[a,1],[b,2]]).`
  - Результат: `Y = [1,2]`.
- Если списки разной длины, предикат не должен успешно унифицироваться (в данной реализации он требует равной длины).

## Задание 3. `contains_duplicates(List)`

**Условие:** истинно, если список содержит дубликаты.

**Как работает:**
1. Разделяем список на `Prefix + [X | Tail]`.
2. Проверяем, является ли `X` членом `Tail`.
3. Если да — дубликат найден, `!` прекращает поиск.

**Примеры и поведение с переменными:**
- `?- contains_duplicates([a,b,a,b,c,c,a]).` — да, успешный ответ.
- `?- contains_duplicates([a,b,c]).` — нет, неудача.
- Если в списке есть переменные, поведение становится недетерминированным:
  - `?- contains_duplicates([X, a, X]).` — успешно (X может быть тем же атомом), но также может оставаться с незвязанной переменной в зависимости от вызовов.
  - `?- contains_duplicates([A,B,C])` — может вернуть `false`, так как без конкретных значений нельзя доказать существование двух одинаковых элементов; при наличии переменных предикат использует `append` и `member`, что может породить решения с конкретными привязками переменных в процессе поиска.

Замечание: тестирование с переменными важно — поведение зависит от порядка генерации вызовов и наличия отсечения (`!`) в определении.

## Задание 4. `my_flatten(Nested, Flat)`

**Условие:** "разгладить" вложенные списки.

**Пример:**
- `?- my_flatten([a, [[b], c], [[d]]], X).`
  - Результат: `X = [a, b, c, d]`.

**Как работает:**
1. Если голова `H` — список, рекурсивно разглаживаем её и хвост.
2. Если `H` — атом, добавляем его в результат и разглаживаем хвост.
3. Базовый случай: пустой список.

## Задание 5. `gray(L, Code)`

**Условие:** генерация кода Грея для N бит (N = длина списка L).

**Пример:**
- `?- gray([0], Code).` → `Code = [[0],[1]]`.
- `?- gray([0,0], Code).` → `Code = [[0,0],[0,1],[1,1],[1,0]]`.

**Как работает:**
1. Для 0 бит — код `[[]]`.
2. Для N бит: берём код для N-1 бит.
3. К оригиналу дописываем 0, к перевёрнутой копии — 1.
4. Объединяем списки.
