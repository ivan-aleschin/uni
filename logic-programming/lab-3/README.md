## Задание 1. `split(Tree, X) -> {TreeLT, TreeGT}`

**Условие:** разделить BST на два дерева: элементы меньше X и элементы больше X.

**Представление дерева:** `nil | {node, Value, Left, Right}`

**Как работает (рекурсивный split без конвертации в список):**
1. Пустое дерево → `{nil, nil}`
2. Если `V < X`: весь левый сын `L` точно < X (свойство BST), он целиком идёт в LT. Правый сын `R` нужно разделить по X. Результат: `LT = {node, V, L, RLT}`, `GT = RGT`
3. Если `V > X`: весь правый сын `R` точно > X, он идёт в GT. Левый сын делится. Результат: `LT = LLT`, `GT = {node, V, LGT, R}`
4. Если `V =:= X`: сам узел исключается. `LT = L` (весь левый сын < X), `GT = R` (весь правый > X)

```
split([1,3,4,5,6,7,8], 5)  — дерево с корнем 5
  V=5 =:= X=5 → LT=поддерево(1,3,4), GT=поддерево(6,7,8)

split([1,3,4,5,6,7,8], 3)  — корень 5
  V=5 > X=3 → делим левое поддерево [1,3,4] по X=3
    V=3 =:= X=3 → LT=[1], GT=[4]
  GT = {node, 5, [4], [6,7,8]}
  LT = [1]
```

**Описание:** Рекурсивный алгоритм работает за O(log n) для сбалансированного дерева, спускаясь по одной ветке на каждом уровне и «подшивая» поддеревья по пути.

## Задание 2–3. Интерфейс множества и `is_subset`

**Интерфейс:**

| Функция | Сигнатура | Описание |
|---------|-----------|----------|
| `new` | `() -> set` | Пустое множество |
| `add` | `(set, any) -> set` | Добавить элемент |
| `remove` | `(set, any) -> set` | Удалить элемент |
| `member` | `(set, any) -> bool` | Проверить наличие |
| `union` | `(set, set) -> set` | Объединение |
| `intersection` | `(set, set) -> set` | Пересечение |
| `difference` | `(set, set) -> set` | Разность |
| `to_list` | `(set) -> list` | В список |
| `from_list` | `(list) -> set` | Из списка |
| `is_subset` | `(set, set) -> bool` | Является ли подмножеством |
| `size` | `(set) -> int` | Мощность |

**`is_subset(Set1, Set2)`** возвращает `true`, если каждый элемент Set1 есть в Set2:

```
is_subset({2,3}, {1,2,3,4})
  member({1,2,3,4}, 2) = true
  member({1,2,3,4}, 3) = true
  → true

is_subset({1,2,3,4}, {2,3})
  member({2,3}, 1) = false
  → false
```

## Задание 4, Реализация 1: `lab3_set_bst` — множество на BST

**Представление:** двоичное дерево поиска `nil | {node, V, L, R}`.

**Как работает `add`:** идём по дереву как при поиске, вставляем как новый лист, дубликаты игнорируем (третий клоз).

**Как работает `remove`:** находим узел, заменяем его минимальным элементом правого поддерева (in-order successor), затем удаляем этот минимум из правого поддерева.

**Как работают `union`, `intersection`, `difference`:** конвертируем через `to_list` и собираем обратно через `from_list` / list comprehension. Просто и читаемо.

**Как работает `is_subset`:**
```erlang
lists:all(fun(X) -> member(T2, X) end, to_list(T1))
```
Проверяем каждый элемент T1 на присутствие в T2.

## Задание 4, Реализация 2: `lab3_set_list` — множество на отсортированном списке

**Представление:** отсортированный список без дубликатов, например `[1, 3, 5, 7]`.

**Как работает `add`:** вставка с сохранением порядка — ищем позицию и вставляем:
```
add([3,5,7], 4)
  4 > 3 → [3 | add([5,7], 4)]
    4 < 5 → [4, 5, 7]
  → [3, 4, 5, 7]
```

**Как работают `union`, `intersection`, `difference`:** слияние двух отсортированных списков за O(n+m) — как в merge sort. Три паттерна: H1 < H2, H1 > H2, H1 = H2.

```
union([1,3,5], [2,3,4])
  1 < 2 → [1 | union([3,5], [2,3,4])]
    3 > 2 → [2 | union([3,5], [3,4])]
      3 = 3 → [3 | union([5], [4])]
        5 > 4 → [4 | union([5], [])]
          → [5]
        → [4,5]
      → [3,4,5]
    → [2,3,4,5]
  → [1,2,3,4,5]
```

**Как работает `is_subset`:** синхронный обход двух отсортированных списков:
- Если `H1 > H2` — элемент H2 точно не нужен, двигаемся вперёд по S2
- Если `H1 = H2` — элемент найден, идём дальше в обоих
- Если `H1 < H2` — элемент S1 не нашёлся в S2, сразу `false`

## Задание 5. Алгоритмическая сложность

Обозначения: n = |Set1|, m = |Set2|. Для BST предполагается сбалансированное дерево (случай вставки случайных элементов). В худшем случае (вырожденное дерево) O(log n) → O(n).

| Операция | BST | Отсортированный список |
|----------|-----|----------------------|
| `add` | O(log n) | O(n) |
| `remove` | O(log n) | O(n) |
| `member` | O(log n) | O(n) |
| `union` | O(m·log(n+m)) | O(n + m) |
| `intersection` | O(n·log m) | O(n + m) |
| `difference` | O(n·log m) | O(n + m) |
| `is_subset` | O(n·log m) | O(n + m) |
| `from_list` | O(n·log n) | O(n²) |

**Вывод:**
- Список выигрывает на `union`/`intersection`/`difference`/`is_subset` благодаря слиянию за O(n+m)
- BST выигрывает на `add`/`remove`/`member` (O(log n) против O(n))
- Для задач с частыми множественными операциями (пересечения, объединения больших множеств) список эффективнее
- Для задач с частыми одиночными вставками/удалениями/проверками — BST
