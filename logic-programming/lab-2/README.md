## Задание 1. `sum_neg_squares(List)`

**Условие:** вернуть сумму квадратов всех отрицательных чисел в списке.
`sum_neg_squares([-3, a, false, -3, 1])` => `18`

**Как работает:**
1. Базовый случай: пустой список → 0
2. Голова `H` — отрицательное число (`is_number(H), H < 0`): прибавляем `H*H` и рекурсируем по хвосту
3. Всё остальное (атомы, булевы, положительные числа) — пропускаем через паттерн `_`
4. Guards (`when`) позволяют проверять условия прямо в заголовке клоза

```
sum_neg_squares([-3, a, false, -3, 1])
  H=-3, -3 < 0 → 9 + sum_neg_squares([a, false, -3, 1])
    H=a, не число → sum_neg_squares([false, -3, 1])
      H=false, не число → sum_neg_squares([-3, 1])
        H=-3, -3 < 0 → 9 + sum_neg_squares([1])
          H=1, 1 >= 0 → sum_neg_squares([])
            → 0
        → 9
      → 9
    → 9
  → 18
```

**Описание:** Три клоза — пустой список, подходящий элемент и всё остальное. `is_number/1` защищает от атомов и булевых значений, которые не поддерживают сравнение `< 0`.

## Задание 2. `dropwhile(Pred, List)`

**Условие:** отбросить начальные элементы списка, пока `Pred` возвращает `true`.
`dropwhile(fun(X) -> X < 10 end, [1,3,9,11,6])` => `[11, 6]`

**Как работает:**
1. Пустой список → всегда возвращаем `[]`
2. Применяем `Pred` к голове через `case`
3. Если `true` — голова отбрасывается, рекурсируем по хвосту
4. Если `false` — возвращаем `[H|T]` целиком: первый не подошедший элемент и всё после него

```
dropwhile(fun(X) -> X < 10 end, [1, 3, 9, 11, 6])
  H=1,  Pred(1)=true  → отбрасываем, идём дальше
  H=3,  Pred(3)=true  → отбрасываем
  H=9,  Pred(9)=true  → отбрасываем
  H=11, Pred(11)=false → возвращаем [11, 6]
```

**Описание:** `case Pred(H) of` — вызов функции, хранящейся в переменной. Это пример функции высшего порядка: `dropwhile` не знает конкретного условия, она принимает его как аргумент.

## Задание 3. `antimap(ListF, X)`

**Условие:** применить список функций к одному значению `X`.
`antimap([fun(X) -> X+2 end, fun(X) -> X*3 end], 4)` => `[6, 12]`

**Как работает:**
1. Пустой список функций → пустой список результатов
2. Берём первую функцию `F`, вызываем `F(X)`, ставим результат в голову
3. Рекурсируем по оставшимся функциям с тем же `X`

```
antimap([F1, F2], 4)
  F1 = fun(X) -> X + 2 end
  F1(4) = 6
  [6 | antimap([F2], 4)]
    F2 = fun(X) -> X * 3 end
    F2(4) = 12
    [12 | antimap([], 4)]
      → []
    → [12]
  → [6, 12]
```

**Описание:** Это «обратный map» — обычный `lists:map(F, List)` применяет одну функцию к списку элементов, здесь наоборот: применяем список функций к одному элементу.

## Задание 4. `solve(Fun, A, B, Eps)`

**Условие:** найти корень уравнения `Fun(X) = 0` на отрезке `[A, B]` методом бисекции с точностью `Eps`. Гарантируется: `Fun(A) ≤ 0 ≤ Fun(B)`.
`solve(fun(X) -> X*X - 2 end, 0, 2, 0.001)` => `≈ 1.41421`

**Как работает:**
1. Базовый случай: если отрезок `[A,B]` уже меньше `Eps` — возвращаем середину
2. Находим середину: `Mid = (A + B) / 2`
3. Если `Fun(Mid) ≤ 0` — корень в правой половине `[Mid, B]`
4. Если `Fun(Mid) > 0` — корень в левой половине `[A, Mid]`
5. Рекурсия с вдвое меньшим отрезком

```
solve(fun(X) -> X*X - 2 end, 0, 2, 0.001)
  B - A = 2 >= 0.001 → Mid = 1.0
  Fun(1.0) = -1.0 ≤ 0 → solve(Fun, 1.0, 2, 0.001)
    Mid = 1.5, Fun(1.5) = 0.25 > 0 → solve(Fun, 1.0, 1.5, ...)
      Mid = 1.25, Fun(1.25) = -0.4375 ≤ 0 → solve(Fun, 1.25, 1.5, ...)
        ... (продолжается до B - A < 0.001)
  → ≈ 1.41421
```

**Описание:** Метод деления пополам (бисекции). На каждом шаге длина отрезка делится на 2, поэтому число шагов = `ceil(log2((B-A)/Eps))`. Знак `Fun(Mid)` определяет, в какой половине находится смена знака (= корень).

## Задание 5. `for(Init, Cond, Step, Body)`

**Условие:** реализовать аналог цикла `for` из C: `for(I=Init; Cond(I); I=Step(I)) { Body(I) }`.

**Как работает:**
1. Проверяем `Cond(Init)` через `case`
2. Если `false` — цикл завершён, возвращаем атом `ok`
3. Если `true` — выполняем `Body(Init)`, вычисляем следующее `Step(Init)` и рекурсируем

```
for(1, fun(I) -> I =< 5 end, fun(I) -> I+1 end, fun(I) -> print(I) end)
  Cond(1) = true → Body(1) → печатает 1
    for(2, ...)
      Cond(2) = true → Body(2) → печатает 2
        ...
          for(6, ...)
            Cond(6) = false → ok
```

**Описание:** Рекурсия заменяет изменяемое состояние. В Erlang переменные неизменяемы, поэтому «следующее значение I» передаётся аргументом следующего вызова функции, а не записывается в ту же переменную.

## Задание 6. `sort_by(Comparator, List)` — сортировка слиянием

**Условие:** отсортировать список с пользовательским компаратором.
`Comparator(X, Y)` возвращает `less`, `equal` или `greater`.

**Алгоритм: сортировка слиянием (Merge Sort)**

Три вспомогательных предиката:
- `sort_by` — точка входа, делит список, рекурсирует, сливает
- `split(List)` — делит список на два примерно равных, чередуя элементы
- `merge_by(Cmp, L, R)` — сливает два уже отсортированных списка

**Как работает:**
1. Список из 0 или 1 элемента — уже отсортирован (базовые случаи)
2. `split` делит список, чередуя: чётные → левая половина, нечётные → правая
3. Рекурсивно сортируем обе половины
4. `merge_by` сравнивает головы двух списков, берёт меньшую и рекурсирует

```
sort_by(Cmp, [3, 1, 4, 1])
  split → Left=[4,3], Right=[1,1]
  sort_by(Cmp, [4,3])
    split → Left=[4], Right=[3]
    merge_by([4],[3]) → [3,4]
  sort_by(Cmp, [1,1])
    split → Left=[1], Right=[1]
    merge_by([1],[1]) → [1,1]
  merge_by([3,4],[1,1])
    Cmp(3,1)=greater → [1 | merge([3,4],[1])]
      Cmp(3,1)=greater → [1 | merge([3,4],[])]
        → [3,4]
      → [1,3,4]
    → [1,1,3,4]
```

**Описание:** Сложность — O(n log n). `split` через чередование даёт сбалансированное деление без вычисления длины. В `merge_by` если компаратор возвращает `greater` — берём элемент из правого списка; в остальных случаях (`less` или `equal`) — из левого.
