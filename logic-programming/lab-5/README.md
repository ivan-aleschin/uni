## Задание 1. Классификация последовательностей символов

Для каждой строки укажите, является ли она атомом, переменной, сложным термом или не термом, и краткое объяснение.

1. `переменная`  
   - Ответ: атом.  
   - Обоснование: начинается с строчной буквы → в Прологе это атом (если бы это была переменная, имя должно начинаться с прописной буквы или `_`).

2. `'язык Пролог'`  
   - Ответ: атом (кавычечный).  
   - Обоснование: строка в одинарных кавычках — это атом, можно использовать пробелы и заглавные буквы.

3. `Пролог`  
   - Ответ: переменная.  
   - Обоснование: начинается с заглавной буквы → в Прологе это имя переменной.

4. `Альберт(любит, мясо)`  
   - Ответ: не является валидным термом (синтаксическая ошибка в классическом Прологе).  
   - Обоснование: `Альберт` начинается с заглавной — интерпретируется как переменная; переменная не может быть применена к аргументам как функционный символ. Чтобы это был сложный терм, имя функторa должно начинаться со строчной буквы (например, `альберт(...)`) или быть в кавычках (`'Альберт'(...)`).

5. `любит(Альберт, мясо)`  
   - Ответ: сложный терм (compound term).  
   - Обоснование: `любит` — строчная → функтор, аргументы внутри: `Альберт` — переменная (заглавная), `мясо` — атом.

## Задание 2. Унификация — какие сопоставления удадутся

Разбираем пары выражений и выводим, удаётся ли им унифицироваться и какие будут значения переменных.

1. `meal(food(Y), X) = meal(X, drink(Z))`  
   - Анализ: унифицируются верхние операнды (`meal/2`), поэтому нужно унифицировать `food(Y)` с `X` и `X` с `drink(Z)`. Из второй равенства следует `X = drink(Z)`, тогда подстановка в первом даёт `food(Y) = drink(Z)` — разные имена функторов (`food` vs `drink`) → неунифицируемо.  
   - Ответ: неудача (нет решения).

2. `food(bread, X, beer) = food(Y, burger)`  
   - Анализ: слева `food/3`, справа `food/2` — разная арность.  
   - Ответ: неудача.

3. `likes(X, cola) = likes(Y, cola)`  
   - Анализ: одинаковый функтор `likes/2`, второй аргумент одинаков, первый аргумент `X` и `Y` — переменные. Унификация даёт `X = Y`.  
   - Ответ: удаётся; результат: `X = Y` (т.е. обе переменные ссылаются на один объект).

4. `'bread' = bread`  
   - Анализ: кавычечный атом `'bread'` эквивалентен некавычечному `bread`.  
   - Ответ: удаётся; никакие переменные не связываются (истина).

5. `likes(X, food(Y)) = likes(Y, food(Z))`  
   - Анализ: из первого аргумента `X = Y`. Из второго `food(Y) = food(Z)` → по совпадению функторов получается `Y = Z`. В итоге `X = Y = Z`.  
   - Ответ: удаётся; решение: `X = Y, Y = Z` (все три одинаковы).

## Задание 5. Реализация `тётя/2` (aunt/2)

В этом репозитории предикат `aunt/2` уже реализован в `family.pl`. Правило: `X` — тётя `Y`, если `X` — сестра одного из родителей `Y`.

Посмотрим реализацию (фрагмент `family.pl`):

```uni/logic-programming/lab-5/family.pl#L42-46
% Задание 5: Тётя
% X является тётей Y, если X — сестра одного из родителей Y.
aunt(X, Y) :-
    sister(X, Parent),
    parent(Parent, Y).
```

Примеры запросов и ожидаемые результаты по имеющейся БД:

- `?- aunt(A, mary).`  
  Результат: `A = olga.` (в нашей базе `olga` — сестра `ivan`, `ivan` — родитель `mary` → `olga` — тётя `mary`).

- `?- findall(A-N, aunt(A, N), L).`  
  Пример результата: `L = [olga-mary, olga-alex].`

Если вы захотите изменить определение (например, включить тётей по браку — `sister_in_law`), можно дописать соответствующие правила в `family.pl`.

## Задание 6. Собственная база данных (фильмы) — факты и правила

В `lab-5.pl` присутствует простая база фильмов/актёров, минимум три факта и правила, включая рекурсивное правило для обнаружения связей между актёрами.

Фрагмент с фактами и правилами:

```uni/logic-programming/lab-5/lab-5.pl#L1-20
% Задание 6: Фильмы
movie('Inception', 'Christopher Nolan', 2010).
movie('The Dark Knight', 'Christopher Nolan', 2008).
actor('Inception', 'Leonardo DiCaprio').
actor('Inception', 'Tom Hardy').
actor('The Dark Knight', 'Tom Hardy').
actor('The Dark Knight', 'Christian Bale').

worked_together(A1, A2) :- actor(M, A1), actor(M, A2), A1 \= A2.
```

Рекурсивный предикат поиска связей (путь через посредников):

```uni/logic-programming/lab-5/lab-5.pl#L21-40
actors_connected(A1, A2) :- actors_connected(A1, A2, [A1]).
actors_connected(A1, A2, _) :- worked_together(A1, A2).
actors_connected(A1, A2, V) :-
    worked_together(A1, I), \+ member(I, V),
    actors_connected(I, A2, [I|V]).
```

Пояснение:
- `worked_together/2` — базовый факт: два актёра снимались в одном фильме.
- `actors_connected/2` — рекурсивный поиск через промежуточных актёров; список `Visited` предотвращает зацикливание.

Пример запросов:
- `?- worked_together('Tom Hardy', 'Christian Bale').` → `true`.  
- `?- actors_connected('Leonardo DiCaprio', 'Christian Bale').` → `true` (DiCaprio — Tom Hardy — Christian Bale).

## Задание 7. Полёты. Предикат `связаны(X, Y)`

В `lab-5.pl` есть база рейсов и предикат `связаны/2`, который отвечает истинно, если есть путь из города `X` в город `Y` из одного или нескольких перелётов.

Фрагмент:

```uni/logic-programming/lab-5/lab-5.pl#L30-50
% Задание 7: Полёты
есть_рейс(мос,спб).
есть_рейс(мос,ект).
есть_рейс(мос,новосиб).
есть_рейс(спб,новосиб).
есть_рейс(спб,сочи).
есть_рейс(спб,минск).
есть_рейс(ект,сочи).
есть_рейс(сочи,киев).
есть_рейс(минск,новосиб).
есть_рейс(минск,киев).

% 7' : сделать граф неориентированным — предикат рейс/2
рейс(X, Y) :- есть_рейс(X, Y).
рейс(X, Y) :- есть_рейс(Y, X).

% 7'' : DFS с защитой от циклов
связаны(X, Y) :- связаны(X, Y, [X]).
связаны(X, Y, _) :- рейс(X, Y).
связаны(X, Y, V) :-
    рейс(X, Z), \+ member(Z, V),
    связаны(Z, Y, [Z|V]).
```

Пояснения и замечания по оценке:
- 7' (+1 балл) — превращение `есть_рейс/2` в неориентированный граф через `рейс/2` (как в коде выше) — позволяет использовать рейсы в обоих направлениях.
- 7'' (+1 балл) — использование списка посещённых городов `Visited` в `связаны/3` предотвращает бесконечную рекурсию при наличии циклов в графе.

Примеры запросов:
- `?- связаны(мос, сочи).` → `true` (например, `мос -> ект -> сочи` или `мос -> спб -> сочи`).
- Если вы добавите обратные ребра, то для данной БД все города станут связаны друг с другом (из-за `рейс/2`).
