## Задание 5. `aunt(X, Y)`

**Условие:** задать предикат тётя/2 (X — тётя Y).

**Как работает:**
1. Находим родителя `Parent` для ребёнка `Y`.
2. Находим сестру `X` для этого родителя `Parent`.
3. Предикат `sister/2` проверяет, что у `X` и `Parent` общий отец, `X` — женщина и `X` не совпадает с `Parent`.

**Описание:** Логический вывод строится на композиции двух отношений: родительство и родство (сестра).

## Задание 6. База данных фильмов

**Условие:** создать свою БД (минимум 3 факта, 2 правила, одно рекурсивное).

**Как работает (actors_connected):**
1. Базовый случай: актеры `A1` и `A2` снимались в одном фильме (`worked_together`).
2. Рекурсивный шаг: `A1` снимался с посредником `Inter`, который в свою очередь связан с `A2`.
3. Список `Visited` предотвращает зацикливание, если актеры снимались в нескольких общих проектах.

**Описание:** Демонстрирует использование рекурсии для поиска путей в графе связей.

## Задание 7. `связаны(X, Y)`

**Условие:** найти связь между городами (один или более рейсов).

**Как работает:**
1. Предикат `рейс/2` делает граф неориентированным (7').
2. `связаны(X, Y, Visited)` ищет путь из `X` в `Y`.
3. На каждом шаге выбирается промежуточный город `Z`, проверяется, что мы там не были, и поиск продолжается из `Z`.

**Описание:** Реализация поиска в глубину (DFS) с защитой от циклов (7'') для определения достижимости в графе.
