## Задание 1. `ring(N, M)`

**Условие:** создать N процессов в кольце. Сообщение проходит от 1-го к 2-му и далее. После M полных кругов все процессы завершаются. Все события печатаются через `io:format`.

**Идея реализации:**
1. Создаём список PID процессов.
2. Для каждого процесса запоминаем `Next`, а также `First` и `Last`.
3. Запускаем токен от первого процесса.
4. Первый процесс увеличивает счётчик кругов только когда получает токен от последнего.
5. Когда количество кругов достигает M — запускается `stop`, и он проходит по кольцу, завершая все процессы.
6. `ring/2` ждёт сигнала `{ring_done, First}` и возвращает `ok` только после полного завершения кольца.

**Пример:** (PID могут отличаться)
```/dev/null/example.erl#L1-10
ring:ring(3, 2).
Current process is <0.31.0>
Created <0.33.0>
Created <0.34.0>
Created <0.35.0>
<0.33.0> received 0 from <0.31.0>
<0.34.0> received 0 from <0.33.0>
<0.35.0> received 0 from <0.34.0>
<0.33.0> received 0 from <0.35.0>
...
```

## Задание 2. `parent_children`

**Интерфейс:**
- `start(N)` — запускает родителя и N детей
- `send_to_child(I, Msg)` — отправляет сообщение ребенку I
- `stop()` — останавливает родителя (и детей)

**Поведение детей:**
- `stop` → завершение без ошибки
- `die` → падение с ошибкой
- другое → печатается в консоль

**Поведение родителя:**
- родитель зарегистрирован как `parent_children`
- если ребенок умер с ошибкой → перезапускается и выводится сообщение
- если родитель останавливается → всем детям отправляется `stop`

**Пример:**
```/dev/null/example.erl#L1-6
parent_children:start(3).
parent_children:send_to_child(1, hello).
parent_children:send_to_child(2, {msg, from_parent}).
parent_children:send_to_child(3, die).
parent_children:stop().
```

## Задание 3. `par_filter(F, List, Options)`

**Условие:** параллельная фильтрация списка. Результат содержит те же элементы, что и `lists:filter/2`, но порядок может отличаться.

**Options:**
- `{sublist_size, integer()}` — размер подсписков (по умолчанию 1)
- `{processes, integer()}` — число воркеров (по умолчанию = число частей)
- `{timeout, Milliseconds | infinity}` — максимальное время (по умолчанию infinity)

**Идея реализации:**
1. Разбиваем список на подсписки длины `sublist_size`.
2. Создаём пул из `processes` воркеров.
3. Каждый воркер запрашивает задачу, фильтрует подсписок и возвращает результат.
4. Родитель собирает результаты. При `timeout` возвращает уже собранное и останавливает воркеры.

**Пример:**
```/dev/null/example.erl#L1-3
F = fun(X) -> is_integer(X) andalso (X rem 2 =:= 0) end,
par_filter:par_filter(F, [1,2,3,4,5,6,a,false,8], [{sublist_size,2},{processes,3}]).
```
