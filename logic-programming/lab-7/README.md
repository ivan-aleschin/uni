## Задание 1. `triangle(N, T)`

**Условие:** вычислить N-ное треугольное число (сумма 1..N).

**Как работает:**
1. Если `N = 0` — сумма 0.
2. Иначе вычисляем треугольное число для `N-1`.
3. Прибавляем `N` к результату.

**Описание:** Классическая рекурсия для вычисления суммы ряда.

---

## Задание 2. `dot(L1, L2, Res)`

**Условие:** скалярное произведение двух векторов (списков).

**Как работает:**
1. Перемножаем головы списков `H1 * H2`.
2. Рекурсивно вычисляем произведение для хвостов.
3. Суммируем текущее произведение с результатом из рекурсии.

**Описание:** Прямая реализация математической формулы скалярного произведения.

---

## Задание 3. `polynom(Expr)`

**Условие:** проверка многочлена на нормальную форму (степени убывают, коэффициенты не равны 0).

**Как работает:**
1. Предикат `is_term` разбирает одночлен: `C * x^P`, `x^P`, `C * x` или константу.
2. Предикат `polynom_ordered` обходит сумму `A + B`.
3. Проверяет, что степень `A` строго больше степени первого слагаемого в `B`.

**Описание:** Разбор и валидация символьных выражений как древовидных структур.

---

## Задание 4. `combination(List, K, Result)`

**Условие:** найти сочетания по K элементов.

**Как работает:**
1. Клоз 1: берем голову `H` в результат, ищем остальные `K-1` в хвосте.
2. Клоз 2: пропускаем голову `H`, ищем `K` элементов в хвосте.
3. Базовый случай: `K=0` — пустой список.

**Описание:** Комбинаторный алгоритм выбора подмножества элементов.

## Раздел: операторы `=`, `==`, `=:=` и `is` — разъяснения и примеры

Чтобы понять различия между этими операторами, полезно сначала предсказать, как должен ответить Пролог, а затем проверить на практике. Ниже даны короткие правила и примеры запросов с объяснениями ожидаемого поведения.

Общие принципы:
- `=` — операция унификации: пытается сделать два терма одинаковыми, связывая переменные; не выполняет арифметических вычислений.
- `==` — структурное (строгое) сравнение: проверяет, являются ли два терма тождественными без связывания переменных и без вычислений.
- `=:=` — арифметическое сравнение: вычисляет обе стороны как арифметические выражения и сравнивает численные значения; обе стороны должны быть вычислимыми числами (в противном случае будет ошибка инстанцирования/типа).
- `is` — вычисляет правую часть (как арифметическое выражение) и затем унифицирует результат с левой частью; левая часть обычно должна быть переменной или числом (иначе возможны ошибки).

Примеры и объяснения (каждый пример — типичный вид ответа в SWI‑Prolog):

1) `?- X = 1+2.`  
   - Поведение: `X` связывается с термом `1+2` (без вычисления).  
   - Результат: `X = 1+2`.

2) `?- 3 = 1+2.`  
   - Поведение: `=` не вычисляет арифметику; терм `3` не унифицируется с термом `1+2`.  
   - Результат: `false`.

3) `?- 2+1 = 1+2.`  
   - Поведение: структурно термы `2+1` и `1+2` различны (оператор `+` — функтор с аргументами в разном порядке), `=` не вычисляет — значит `false`.  
   - Результат: `false`.

4) `?- X == 1+2.`  
   - Поведение: `==` сравнивает структуры без связывания; если `X` не была связана ранее, то `X` ≠ `1+2`.  
   - Результат: `false`.

5) `?- 3 == 1+2.`  
   - Поведение: структурное сравнение, `3` ≠ `1+2` как структуры.  
   - Результат: `false`.

6) `?- 2+1 == 1+2.`  
   - Поведение: структуры различны → `false`.

7) `?- X =:= 1+2.`  
   - Поведение: `=:=` пытается вычислить обе стороны как числа; если `X` не связана с числом, операция вызовет ошибку инстанцирования (или ошибку выполнения). Поэтому ожидать корректного `true/false` нельзя — возникнет ошибка или предикат завершится с исключением. (В коде тестов это обрабатывается через `catch`/`throws`.)

8) `?- 3 =:= 1+2.`  
   - Поведение: обе стороны вычисляются — `3 =:= 3` → `true`.

9) `?- 2+1 =:= 1+2.`  
   - Поведение: обе стороны вычисляются (2+1 → 3, 1+2 → 3) → `true`.

10) `?- X is 1+2.`  
    - Поведение: правая часть вычисляется (3) и результат унифицируется с `X`.  
    - Результат: `X = 3`.

11) `?- 3 is 1+2.`  
    - Поведение: правая часть вычисляется (3), затем сравнивается с левой (числом 3) — успешно → `true`.

12) `?- 2+1 is 1+2.`  
    - Поведение: левая часть не является переменной или числом (это составной терм); в большинстве реализаций это считается некорректным использованием `is` и приводит к ошибке (либо типовой ошибке, либо ошибке инстанцирования). Поэтому ожидать `true/false` неправильно — будет ошибка.

Замечания по практике
- `=` полезен, когда нужно связать переменную с термом (включая невычисленные арифметические выражения).  
- `is` — когда нужно вычислить арифметическое выражение и записать результат в переменную.  
- `=:=` — для сравнения числовых значений; обе стороны должны быть числовыми выражениями.  
- `==` и `\==` — для строгой проверки идентичности термов без связывания.
