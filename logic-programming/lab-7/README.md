## Перечень реализованных и описанных случаев

1. Triangle:
   - Рекурсивный расчёт треугольного числа N (N натуральное, N >= 0).

2. Dot product:
   - Скалярное произведение двух списков одинаковой длины (реализация предполагает совпадающие длины).

3. Polynom (проверка формы):
   - Распознаются одночлены и многочлены в форме суммы термов `C * x^P`, `x^P`, `C * x`, `x`, `C` где:
     - `C` — числовой коэффициент (number), `C \= 0`.
     - `P` — целая степень (`integer(P)`), `P > 0` (для x^P).
   - Проверяется порядок слагаемых (степени строго убывают).

4. Combination:
   - Генерация всех K‑сочетаний без учёта порядка.

5. Prime factors (доп. задание 5):
   - Для целого N > 1 выполняется разложение на простые множители методом деления (trial division).
   - Результат — список `factor(Prime, Power)` в порядке возрастания простых множителей, например:
     - `prime_factors(315, L)` даёт `L = [factor(3,2), factor(5,1), factor(7,1)]`.
   - В README и в коде описаны случаи:
     - N простое (короткий случай — возвращаем `factor(N,1)`).
     - N делится на текущее предполагаемое простое F; подсчёт кратности осуществляется отдельным блоком.
     - N содержит большие простые множители (обработка остатка при F*F > N).

6. Polynomize (доп. задание 6):
   - Преобразование арифметического выражения в многочлен в нормальной форме (сумма одночленов `C * x^D`, упорядоченных по степени).
   - Поддерживаемые синтаксические случаи (реализованные и протестированные):
     - Константы (числа).
     - Переменная `x`.
     - Возведение в степень `x^N`, где `N` — неотрицательное целое.
     - Сложение `A + B`.
     - Умножение `A * B`, включая умножение на числовой коэффициент (обе стороны).
     - Унарный минус (`-A`), масштабирование коэффициентов.
     - Комбинации выше (например, `(x + x)*(x + 1)`).
   - В реализации выражение сначала конвертируется в набор пар (степень, коэффициент), затем объединяются одинаковые степени и строится финальный синтаксический терм (например, `2*x^2 + 2*x`).
   - В README указаны граничные/вспомогательные правила:
     - Не поддерживаются произвольные символы отличные от `x` (они трактуются как нулевой вклад в простом fallback варианте — можно расширить при необходимости).
     - Степени должны быть целыми неотрицательными; коэффициенты — числовые.

## Дополнительное задание 5. `prime_factors(Num, Factors)`

Что реализовано (коротко):
- Метод: пробное деление (trial division) с подсчётом кратности каждого простого делителя.
- Специальные случаи, которые описаны и покрыты тестами:
  - N простое (быстрая ветвь при условии F*F > N).
  - N делится на минимальный делитель F — подсчитывается степень F (циклом или рекурсивно), затем продолжаем с остатком.
  - N = 1 — возвращаем пустой список.
- Пример:
  - `prime_factors(315, L)` → `L = [factor(3,2), factor(5,1), factor(7,1)]`.

Комментарии к блокам кода в `lab-7.pl` (русские):
- `prime_factors/2` — основной интерфейс: проверка корректности входа и запуск аккумулятора.
- `pf_acc/3` — аккумулируем пары `factor(Prime,Power)` рекурсивно, продвигая делитель `F`.
- Ветка `F * F > N` — оптимизация: если текущий делитель превысил sqrt(N), то остаток N — простое, добавляем его с кратностью 1.
- `count_power/5` — подсчитывает кратность делителя F в N (сколько раз F входит в факторизацию) и возвращает остаток.

## Дополнительное задание 6. `polynomize(Expr, Poly)`

Что реализовано (коротко):
- Преобразование произвольного арифметического выражения (ограниченного операциями +, *, ^, унарный минус, числовые коэффициенты и символ `x`) в нормализованный полином.
- Стратегия реализации:
  1. `expr_to_pairs/2` — разбирает выражение и переводит его в «сырые» пары (степень, коэффициент) вида `(D,C)` (временный формат).
     - Обрабатываются: числа, `x`, `x^N`, `A + B`, `A * B`, `C * E` (коэффициент числом), унарный минус.
  2. `pairs_to_deg/2` — нормализует представление к виду `deg(D,C)` (внутренний удобный формат).
  3. `combine_pairs/2` — складывает коэффициенты по одинаковым степеням, отбрасывает нулевые коэффициенты и упорядочивает степени по убыванию.
  4. `pairs_to_term/2` — собирает окончательное синтаксическое выражение многочлена (`C * x^D + ...`) из нормализованного списка.
- Примеры:
  - `(x + x)*(x + 1)` → сначала раскладывается на пары, затем объединяется → `2*x^2 + 2*x`.
