## Теория: Оконные функции, подзапросы и CTE

### Оконные функции (Window Functions)
Вычисляют значение для каждой строки относительно «окна» — набора связанных строк, **не сворачивая** результат.

```sql
функция() OVER (
    [PARTITION BY столбец]  -- разбить на группы
    [ORDER BY столбец]       -- порядок внутри окна
    [ROWS/RANGE ...]         -- размер рамки (опционально)
)
```

| Функция | Назначение |
|---|---|
| `ROW_NUMBER()` | Уникальный номер строки внутри раздела |
| `RANK()` | Ранг с пропуском (1,1,3) при равных значениях |
| `DENSE_RANK()` | Плотный ранг без пропуска (1,1,2) |
| `LAG(x, n)` | Значение из строки **на n позиций выше** |
| `LEAD(x, n)` | Значение из строки **на n позиций ниже** |
| `SUM(x) OVER (...)` | Нарастающая/оконная сумма |
| `COUNT(*) OVER (...)` | Количество в окне |

### Разница OVER() без параметров vs с PARTITION BY
```sql
-- Сумма по всей таблице (одно значение для всех строк)
SUM(price) OVER ()

-- Сумма по каждому рейсу отдельно
SUM(price) OVER (PARTITION BY flight_id)
```

### Подзапросы
```sql
-- В SELECT — скалярный подзапрос (возвращает одно значение)
SELECT flight_id, (SELECT COUNT(*) FROM segments WHERE flight_id = f.flight_id) AS cnt FROM flights f;

-- В FROM — inline-представление
SELECT * FROM (SELECT ...) AS sub WHERE sub.x > 10;

-- В WHERE — полусоединение
SELECT * FROM tickets WHERE ticket_no IN (SELECT ticket_no FROM segments WHERE price > 5000);
```

### Когда что использовать
- **GROUP BY**: когда нужен один итоговый ряд на группу.
- **OVER()**: когда нужны детальные строки + агрегат по окну одновременно.
- **CTE**: когда подзапрос используется несколько раз или для читаемости.
