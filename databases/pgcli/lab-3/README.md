# Лабораторная работа 3

## Задание
**Реализовать изменения структуры. Продемонстрировать работу скрипта, меняющего структуру и преобразующего данные (если это необходимо). Утрата данных не допускается!**

В этой работе мы на практике реализуем спроектированную схему из Лабораторной работы 2 (Вариант 1) с помощью SQL-скриптов. Структура БД меняется без удаления или перезаписи существующих таблиц, поэтому исходные данные (билеты, рейсы и расписание) остаются в полной сохранности.

---

## Структура файлов в папке `lab-3`

1. `01_schema_update.sql` — основной DDL-скрипт. Вносит изменения: добавляет нужные колонки в существующие таблицы `airplanes_data`, `airports_data`, `flights`, создает таблицу для заблокированных мест и обновляет представления.
2. `02_demonstration.sql` — DML-скрипт демонстрации. Заполняет новые колонки моковыми данными (задает ВПП для Внуково и Boeing 777, блокирует два сломанных кресла на случайном рейсе, перенаправляет один рейс в резервный порт). Затем делает `SELECT`, чтобы показать, что всё работает.

---

## Пайплайн запуска в `pgcli` (Демонстрация работы)

Убедитесь, что вы находитесь в папке `lab-3`.

1. **Запустите ваш клиент:**
```bash
pg demo
```

2. **Примените изменения структуры:**
Внутри `pgcli` выполните:
```sql
\i 01_schema_update.sql
```
*Вы увидите сообщения `ALTER TABLE`, `CREATE VIEW` и `CREATE TABLE`. Никаких ошибок или потерь данных не будет.*

3. **Проверьте новые изменения (необязательно):**
Можете проверить, что у аэропортов появилось новое поле:
```sql
\d airports
```

4. **Запустите демонстрацию данных:**
Выполните скрипт, который добавит тестовые данные и сразу выведет их на экран:
```sql
\i 02_demonstration.sql
```
В терминале вы увидите обновленные записи (информацию о ВПП, заблокированные кресла "Сломано кресло" и рейс, который был перенаправлен в SVO из-за метеоусловий).
