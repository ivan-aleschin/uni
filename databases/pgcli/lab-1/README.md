## Общие правила
Лабораторные проводятся с СУБД PostgreSQL, БД demo.
Описание БД demo - https://postgrespro.ru/education/demodb

## Как скачать и настроить базу данных

Чтобы скачать и загрузить базу данных в PostgreSQL (версия за 3 месяца, архив ~133 МБ -> ~1.3 ГБ на диске), выполните следующие команды в терминале:

```bash
# Скачать маленькую версию
curl -O https://edu.postgrespro.ru/demo-20250901-3m.sql.gz

# Загрузить в PostgreSQL
gunzip -c demo-20250901-3m.sql.gz | psql -h localhost -U $USER

# Удалить архив после успешной загрузки
rm demo-20250901-3m.sql.gz
```

Затем вы можете проверить подключение, используя алиас `pg` (который запускает `pgcli`), и подключиться к базе данных:
```sql
\c demo
SET search_path = bookings;
```

## Структура базы данных

Схема `bookings` содержит следующие основные таблицы и представления:

- `airplanes` (представление) / `airplanes_data` (таблица): Данные о самолетах (код, модель, дальность, скорость).
- `airports` (представление) / `airports_data` (таблица): Данные об аэропортах (код, название, город, страна, координаты, часовой пояс).
- `boarding_passes`: Посадочные талоны, привязанные к билетам и рейсам.
- `bookings`: Данные о бронированиях (номер брони, дата, общая сумма).
- `flights`: Данные о расписании и фактических рейсах (id, маршрут, статус, время вылета/прилета).
- `routes`: Данные о маршрутах между аэропортами.
- `seats`: Доступные места для различных моделей самолетов.
- `segments`: Полетные сегменты, связанные с билетами.
- `tickets`: Информация о билетах пассажиров.
- `timetable` (представление): Удобное представление, объединяющее информацию о рейсах, маршрутах и аэропортах.

## Теория: SQL SELECT и JOIN

### Структура запроса
```sql
SELECT <столбцы>
FROM <таблица>
[JOIN <другая_таблица> ON <условие>]
[WHERE <фильтр>]
[GROUP BY <столбцы>]
[HAVING <фильтр_по_группам>]
[ORDER BY <столбцы>]
[LIMIT <n>];
```

### Типы JOIN
| Тип | Результат |
|---|---|
| `INNER JOIN` (= `JOIN`) | только строки с совпадением в обеих таблицах |
| `LEFT JOIN` | все строки слева + совпадения справа (NULL если нет) |
| `RIGHT JOIN` | все строки справа + совпадения слева |
| `FULL JOIN` | все строки из обеих таблиц |

### GROUP BY + агрегатные функции
- `COUNT(*)` — количество строк
- `SUM(x)`, `AVG(x)`, `MIN(x)`, `MAX(x)` — по значению столбца
- `HAVING` — фильтр **после** агрегации (в отличие от `WHERE` — до)

### CTE (Common Table Expression)
```sql
WITH cte_name AS (
    SELECT ...
)
SELECT * FROM cte_name;
```
CTE — именованный подзапрос, вычисляется **один раз** и переиспользуется.

### Полезные функции в demo DB
- `lang()` — возвращает текущий язык (`'ru'` или `'en'`) из `bookings.lang`
- `airport_name ->> 'ru'` — извлечь строку из JSONB по ключу
- `EXTRACT(DOW FROM timestamp)` — день недели (0=Вс, 1=Пн, …, 6=Сб)
- `tstzrange @> timestamp` — проверить, входит ли момент времени в диапазон
