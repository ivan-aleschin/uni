## pgcli — основные команды (шпаргалка)

Здесь собраны все основные команды для работы в `pgcli`, которые пригодятся во всех лабораторных работах:

**Подключение и навигация:**
- `\l` — список всех баз данных
- `\c demo` — подключиться к базе `demo`
- `\dn` — список схем в текущей базе
- `SET search_path = bookings;` — установить схему по умолчанию (чтобы не писать `bookings.flights`)
- `\q` — выход из pgcli

**Просмотр структуры (метаданные):**
- `\dt` — список таблиц в текущей схеме
- `\dv` — список представлений (views)
- `\df` — список функций
- `\d table_name` — показать структуру конкретной таблицы или представления (колонки, типы, индексы)
- `\d+ table_name` — структура с дополнительными подробностями (комментарии, размер)

**Выполнение скриптов и форматирование:**
- `\i 01.sql` — выполнить SQL-скрипт из файла (очень полезно для лаб)
- `\x` — включить/выключить расширенный вывод (удобно, когда таблица слишком широкая и не влезает в экран, выводит записи вертикально)
- `\timing` — включить/выключить отображение времени выполнения каждого запроса
- `\e` — открыть текущий запрос в текстовом редакторе (`$EDITOR`) для удобного изменения

**Горячие клавиши:**
- `F3` или `Ctrl+E` — быстро открыть внешний редактор
- `↑↓` — навигация по истории запросов
- `Tab` — автодополнение таблиц, столбцов, ключевых слов

> 💡 **Совет:** Многострочный режим включён по умолчанию — чтобы запрос выполнился, обязательно ставь `;` в конце и жми `Enter`.

## Установка демобазы

В твоём терминале (не внутри pgcli, а в обычном fish):

```bash
# Скачать маленькую версию (3 месяца, ~133 МБ → ~1.3 ГБ на диске)
curl -O https://edu.postgrespro.ru/demo-20250901-3m.sql.gz

# Загрузить в PostgreSQL
gunzip -c demo-20250901-3m.sql.gz | psql -h localhost -U $USER

# Удалить архив после загрузки
rm demo-20250901-3m.sql.gz
```

Для проверки работы используй алиас `pg` (запускает pgcli), и выполни:
```sql
\c demo
SET search_path = bookings;
SELECT bookings.version();
```

Должно ответить что-то вроде `PostgresPro 2025-09-01`.

**Стандартный пайплайн выполнения скриптов:**
1. Переходишь в папку с лабой: `cd lab-1`
2. Запускаешь клиент сразу с нужной базой: `pg demo`
3. Внутри pgcli выполняешь нужный скрипт командой `\i`:
```sql
\i 01.sql
```
Каждый `.sql` файл уже должен содержать `SET search_path = bookings;`, так что запросы выполнятся корректно.

> ⚠️ Минимальная поддерживаемая версия — PostgreSQL 15, у тебя 18.2 — всё ок.

## Структура БД (схема bookings)

В базе данных `demo` основные таблицы находятся в схеме `bookings`:

*   **`airplanes_data`** (представление `airplanes`) — Данные о самолетах (код, модель, дальность, скорость).
*   **`airports_data`** (представление `airports`) — Данные об аэропортах (код, название, город, страна, координаты, часовой пояс).
*   **`boarding_passes`** — Посадочные талоны (привязаны к билетам и рейсам).
*   **`bookings`** — Бронирования (номер брони, дата, общая сумма).
*   **`flights`** — Данные о рейсах (статус, расписание, фактическое время вылета/прилета).
*   **`routes`** — Маршруты между аэропортами.
*   **`seats`** — Места в самолетах.
*   **`segments`** — Полетные сегменты в билетах.
*   **`tickets`** — Билеты пассажиров.
*   **`timetable`** (представление) — Удобная сводка расписания, объединяющая рейсы, маршруты и аэропорты.

## Структура проекта

Актуальная структура папок и файлов для лабораторных работ:

```text
pgcli/
├── README.md        # Общая шпаргалка и инструкции (этот файл)
└── lab-1/
    ├── README.md    # Задания и описание конкретной лабораторной
    ├── 01.sql       # SQL-скрипт с решением 1-го запроса
    ├── 02.sql       # SQL-скрипт с решением 2-го запроса
    ...
    └── 12.sql       # SQL-скрипт с решением 12-го запроса
```

**Почему так удобно работать:**
- SQL-файлы редактор подсвечивает как код — работает автодополнение и форматирование.
- Каждый файл можно запустить напрямую внутри `pgcli` (например: `\i 01.sql`).
- Преподавателю легко проверять каждый запрос отдельно, переключаясь между файлами в редакторе.
