
# Конспект по SQL и БД для контрольной работы

## Часть 1: Теоретические вопросы (короткие определения)

### **База данных (БД)**
Организованная совокупность структурированных данных, хранящихся в электронном виде и предназначенных для эффективного доступа и управления.

### **Система управления базами данных (СУБД)**
Программное обеспечение для создания, управления и обеспечения доступа к базам данных (примеры: MySQL, PostgreSQL, Oracle).

### **Модель данных**
Формальное описание структуры данных, связей между ними и ограничений целостности (реляционная, иерархическая, сетевая модели).

### **Отношение**
Таблица в реляционной БД, состоящая из строк (кортежей) и столбцов (атрибутов).

### **Кортеж**
Строка таблицы, представляющая один экземпляр данных с конкретными значениями атрибутов.

### **Атрибут**
Столбец таблицы, характеризующий определенное свойство или поле данных.

### **Первичный ключ (Primary Key)**
Атрибут или набор атрибутов, однозначно идентифицирующий каждую строку в таблице и не допускающий NULL. 

### **Внешний ключ (Foreign Key)**
Атрибут в одной таблице, который ссылается на первичный ключ другой таблицы, обеспечивая связь между таблицами.

### **Транзакция**
Последовательность операций с БД, выполняемая как единое целое (либо все выполняется, либо ничего — принцип ACID).

### **Блокировка**
Механизм управления одновременным доступом к данным, предотвращающий конфликты при параллельной работе нескольких транзакций.

### **Операторы SQL описания данных (DDL)**
Команды для определения структуры БД: CREATE, ALTER, DROP, TRUNCATE.

### **Операторы SQL манипулирования данными (DML)**
Команды для работы с данными: SELECT, INSERT, UPDATE, DELETE.

### **Операторы SQL управления транзакциями (TCL)**
Команды для управления транзакциями: BEGIN/START TRANSACTION, COMMIT, ROLLBACK, SAVEPOINT.

### **Операторы SQL управления доступом к данным (DCL)**
Команды для управления правами доступа: GRANT (предоставление прав), REVOKE (отзыв прав).

### **Нормализация**
Процесс организации данных в БД для уменьшения избыточности и устранения аномалий при операциях вставки, обновления и удаления.

### **Нормальная форма**
Уровень нормализации отношения, определяющий степень устранения избыточности (1НФ, 2НФ, 3НФ, BCNF и т.д.).

## Часть 2: Основные SQL команды и концепции

### **DDL — Data Definition Language**

```sql
-- Создание таблицы
CREATE TABLE Студенты (
    ID INT PRIMARY KEY,
    Имя VARCHAR(100),
    Группа VARCHAR(10),
    Возраст INT
);

-- Изменение структуры таблицы
ALTER TABLE Студенты ADD Email VARCHAR(100);
ALTER TABLE Студенты DROP COLUMN Возраст;
ALTER TABLE Студенты MODIFY Имя VARCHAR(150);

-- Удаление таблицы
DROP TABLE Студенты;

-- Очистка таблицы (удаление всех данных)
TRUNCATE TABLE Студенты;
```

### **DML — Data Manipulation Language**

```sql
-- Вставка данных
INSERT INTO Студенты (ID, Имя, Группа) 
VALUES (1, 'Иванов', 'ПИ-101');

-- Обновление данных
UPDATE Студенты 
SET Группа = 'ПИ-102' 
WHERE ID = 1;

-- Удаление данных
DELETE FROM Студенты 
WHERE ID = 1;

-- Выборка данных
SELECT * FROM Студенты;
SELECT Имя, Группа FROM Студенты WHERE Группа = 'ПИ-101';
```

### **SELECT — подробно**

#### Базовый синтаксис:
```sql
SELECT [DISTINCT] столбцы
FROM таблица
[WHERE условие]
[GROUP BY столбцы]
[HAVING условие_для_групп]
[ORDER BY столбцы [ASC|DESC]]
[LIMIT количество];
```

#### Операторы в WHERE:
```sql
-- Сравнение
WHERE Возраст > 18
WHERE Имя = 'Иванов'
WHERE Оценка >= 4

-- Логические операторы
WHERE Возраст > 18 AND Группа = 'ПИ-101'
WHERE Возраст < 20 OR Возраст > 25
WHERE NOT Группа = 'ПИ-102'

-- Диапазон
WHERE Возраст BETWEEN 18 AND 25

-- Список значений
WHERE Группа IN ('ПИ-101', 'ПИ-102', 'ПИ-103')

-- Поиск по шаблону
WHERE Имя LIKE 'Ива%'  -- начинается с "Ива"
WHERE Имя LIKE '%ов'   -- заканчивается на "ов"
WHERE Имя LIKE '%ван%' -- содержит "ван"

-- Проверка на NULL
WHERE Email IS NULL
WHERE Email IS NOT NULL
```

### **JOIN — объединение таблиц**

```sql
-- INNER JOIN (только совпадающие записи)
SELECT Студенты.Имя, Оценки.Оценка
FROM Студенты
INNER JOIN Оценки ON Студенты.ID = Оценки.ID_Студента;

-- LEFT JOIN (все из левой + совпадающие из правой)
SELECT Студенты.Имя, Оценки.Оценка
FROM Студенты
LEFT JOIN Оценки ON Студенты.ID = Оценки.ID_Студента;

-- RIGHT JOIN (все из правой + совпадающие из левой)
SELECT Студенты.Имя, Оценки.Оценка
FROM Студенты
RIGHT JOIN Оценки ON Студенты.ID = Оценки. ID_Студента;

-- FULL OUTER JOIN (все из обеих таблиц)
SELECT Студенты.Имя, Оценки.Оценка
FROM Студенты
FULL OUTER JOIN Оценки ON Студенты.ID = Оценки.ID_Студента;

-- Множественные JOIN
SELECT С.Имя, П.Название, О.Оценка
FROM Студенты С
JOIN Оценки О ON С.ID = О.ID_Студента
JOIN Предметы П ON О.ID_Предмета = П.ID;
```

### **GROUP BY и агрегатные функции**

```sql
-- Агрегатные функции
COUNT(*)         -- количество строк
COUNT(столбец)   -- количество непустых значений
SUM(столбец)     -- сумма
AVG(столбец)     -- среднее значение
MAX(столбec)     -- максимум
MIN(столбец)     -- минимум

-- Группировка
SELECT Группа, COUNT(*) AS Количество
FROM Студенты
GROUP BY Группа;

SELECT Группа, AVG(Возраст) AS Средний_возраст
FROM Студенты
GROUP BY Группа;

-- HAVING (условие для групп)
SELECT Группа, COUNT(*) AS Количество
FROM Студенты
GROUP BY Группа
HAVING COUNT(*) > 20;
```

### **Подзапросы (Subqueries)**

```sql
-- В WHERE
SELECT Имя 
FROM Студенты
WHERE ID IN (SELECT ID_Студента FROM Оценки WHERE Оценка = 5);

-- В FROM
SELECT Группа, AVG(Средняя) 
FROM (
    SELECT Группа, AVG(Оценка) AS Средняя
    FROM Студенты JOIN Оценки ON Студенты.ID = Оценки.ID_Студента
    GROUP BY Группа, Студенты.ID
) AS Подзапрос
GROUP BY Группа;

-- Скалярный подзапрос
SELECT Имя 
FROM Студенты
WHERE Возраст > (SELECT AVG(Возраст) FROM Студенты);

-- EXISTS
SELECT Имя
FROM Студенты С
WHERE EXISTS (
    SELECT 1 FROM Оценки О 
    WHERE О.ID_Студента = С.ID AND О.Оценка = 5
);
```

### **Операторы множеств**

```sql
-- UNION (объединение без дубликатов)
SELECT Имя FROM Студенты_Группа1
UNION
SELECT Имя FROM Студенты_Группа2;

-- UNION ALL (объединение с дубликатами)
SELECT Имя FROM Студенты_Группа1
UNION ALL
SELECT Имя FROM Студенты_Группа2;

-- INTERSECT (пересечение)
SELECT Имя FROM Студенты_Группа1
INTERSECT
SELECT Имя FROM Студенты_Группа2;

-- EXCEPT/MINUS (разность)
SELECT Имя FROM Студенты_Группа1
EXCEPT
SELECT Имя FROM Студенты_Группа2;
```

### **TCL — Transaction Control Language**

```sql
-- Начало транзакции
START TRANSACTION;
-- или
BEGIN;

-- Фиксация изменений
COMMIT;

-- Откат изменений
ROLLBACK;

-- Точка сохранения
SAVEPOINT точка1;
ROLLBACK TO точка1;
```

## Часть 3: Решение примеров из задания

### **Структура БД (предполагаемая)**

```sql
Студенты (ID, Имя, Группа)
Предметы (ID, Название)
Оценки (ID_Студента, ID_Предмета, Оценка)
```

### **Задача 1:** Имя студента, имеющего "5" по Математике и Физике, и не имеющего "5" по Химии

#### Решение 1 (через подзапросы):
```sql
SELECT DISTINCT С.Имя
FROM Студенты С
WHERE С.ID IN (
    SELECT О.ID_Студента 
    FROM Оценки О
    JOIN Предметы П ON О.ID_Предмета = П.ID
    WHERE П.Название = 'Математика' AND О.Оценка = 5
)
AND С.ID IN (
    SELECT О.ID_Студента 
    FROM Оценки О
    JOIN Предметы П ON О.ID_Предмета = П. ID
    WHERE П.Название = 'Физика' AND О.Оценка = 5
)
AND С.ID NOT IN (
    SELECT О.ID_Студента 
    FROM Оценки О
    JOIN Предметы П ON О.ID_Предмета = П.ID
    WHERE П.Название = 'Химия' AND О.Оценка = 5
);
```

#### Решение 2 (через JOIN и GROUP BY):
```sql
SELECT С.Имя
FROM Студенты С
JOIN Оценки О ON С.ID = О.ID_Студента
JOIN Предметы П ON О. ID_Предмета = П.ID
WHERE П.Название IN ('Математика', 'Физика') AND О.Оценка = 5
GROUP BY С.ID, С.Имя
HAVING COUNT(DISTINCT П.Название) = 2
AND С.ID NOT IN (
    SELECT О2.ID_Студента 
    FROM Оценки О2
    JOIN Предметы П2 ON О2.ID_Предмета = П2.ID
    WHERE П2.Название = 'Химия' AND О2.Оценка = 5
);
```

### **Задача 2:** Группа, Средняя оценка по Математике. Только группы с числом студентов больше 20.

```sql
SELECT С.Группа, AVG(О.Оценка) AS Средняя_оценка
FROM Студенты С
JOIN Оценки О ON С.ID = О.ID_Студента
JOIN Предметы П ON О.ID_Предмета = П.ID
WHERE П.Название = 'Математика'
  AND С.Группа IN (
      SELECT Группа
      FROM Студенты
      GROUP BY Группа
      HAVING COUNT(*) > 20
  )
GROUP BY С.Группа;
```

#### Альтернативное решение (с двумя агрегациями):
```sql
SELECT С.Группа, AVG(О.Оценка) AS Средняя_оценка
FROM Студенты С
JOIN Оценки О ON С.ID = О.ID_Студента
JOIN Предметы П ON О.ID_Предмета = П.ID
WHERE П.Название = 'Математика'
GROUP BY С.Группа
HAVING С.Группа IN (
    SELECT Группа
    FROM Студенты
    GROUP BY Группа
    HAVING COUNT(DISTINCT ID) > 20
);
```

## Часть 4: Типовые паттерны решения задач

### **Паттерн 1: "Студенты, у которых есть оценка X по предмету Y"**
```sql
SELECT DISTINCT С.Имя
FROM Студенты С
JOIN Оценки О ON С.ID = О.ID_Студента
JOIN Предметы П ON О.ID_Предмета = П.ID
WHERE П. Название = 'Математика' AND О.Оценка = 5;
```

### **Паттерн 2: "Студенты, у которых НЕТ оценки X по предмету Y"**
```sql
SELECT С.Имя
FROM Студенты С
WHERE С.ID NOT IN (
    SELECT О.ID_Студента
    FROM Оценки О
    JOIN Предметы П ON О.ID_Предмета = П.ID
    WHERE П.Название = 'Химия' AND О.Оценка = 5
);
```

### **Паттерн 3: "Студенты с оценками по ВСЕМ из списка предметов"**
```sql
SELECT С.Имя
FROM Студенты С
JOIN Оценки О ON С.ID = О.ID_Студента
JOIN Предметы П ON О.ID_Предмета = П.ID
WHERE П.Название IN ('Математика', 'Физика', 'Химия')
GROUP BY С.ID, С.Имя
HAVING COUNT(DISTINCT П.Название) = 3;
```

### **Паттерн 4: "Средние значения по группам с фильтром"**
```sql
SELECT Группа, AVG(значение) AS Среднее
FROM таблица
WHERE условие
GROUP BY Группа
HAVING условие_для_группы;
```

### **Паттерн 5: "Найти MAX/MIN с дополнительной информацией"**
```sql
SELECT С.Имя, О.Оценка
FROM Студенты С
JOIN Оценки О ON С. ID = О.ID_Студента
WHERE О.Оценка = (SELECT MAX(Оценка) FROM Оценки);
```

### **Паттерн 6: "Студенты, сдавшие больше N предметов"**
```sql
SELECT С.Имя, COUNT(DISTINCT О.ID_Предмета) AS Количество_предметов
FROM Студенты С
JOIN Оценки О ON С.ID = О.ID_Студента
GROUP BY С.ID, С. Имя
HAVING COUNT(DISTINCT О.ID_Предмета) > 3;
```

## Часть 5: Важные моменты и частые ошибки

### **Ключевые правила:**

1. **DISTINCT** — убирает дубликаты из результата
2. **GROUP BY** — обязательно включает все не-агрегированные столбцы из SELECT
3. **HAVING** — только после GROUP BY, для фильтрации групп
4. **WHERE** — фильтрация до группировки
5. **ORDER BY** — всегда в конце запроса

### **Порядок выполнения SQL:**
```
FROM → WHERE → GROUP BY → HAVING → SELECT → DISTINCT → ORDER BY → LIMIT
```

### **Порядок написания SQL:**
```
SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT
```

### **Частые ошибки:**

❌ **Неправильно:**
```sql
SELECT Имя, COUNT(*)
FROM Студенты
WHERE COUNT(*) > 5;  -- WHERE не работает с агрегатными функциями! 
```

✅ **Правильно:**
```sql
SELECT Имя, COUNT(*)
FROM Студенты
GROUP BY Имя
HAVING COUNT(*) > 5;
```

---

❌ **Неправильно:**
```sql
SELECT Группа, Имя, COUNT(*)
FROM Студенты
GROUP BY Группа;  -- Имя не в GROUP BY!
```

✅ **Правильно:**
```sql
SELECT Группа, Имя, COUNT(*)
FROM Студенты
GROUP BY Группа, Имя;
```

---

### **Работа с NULL:**

```sql
-- NULL не равен ничему, даже NULL
WHERE значение IS NULL      -- правильно
WHERE значение = NULL       -- неправильно! 

-- Функции для работы с NULL
COALESCE(значение, 0)       -- возвращает первое не-NULL значение
IFNULL(значение, 0)         -- MySQL
NVL(значение, 0)            -- Oracle
```

- ✅ Определить, какие таблицы нужны

- ✅ Проверить условия WHERE
