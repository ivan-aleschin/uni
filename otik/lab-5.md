

## 📊 **RLE (Вариант 2) - Run-Length Encoding**

### **Основная идея:**
Сжатие последовательностей одинаковых символов с использованием **префикса p**

### **Форматы кодирования:**
1. **L>4 одинаковых c≠p**: `(p:8, L-3:8, c:8)` - 3 байта
2. **L>2 символов p**: `(p:8, L-1:8, p:8)` - 3 байта  
3. **Одиночный p**: `(p:8, 0:8)` - 2 байта
4. **Обычные символы**: кодируются как есть (1 байт)
### **Ключевые параметры:**
- **Префикс p** = 0x00 (выбирается на основе частот символов)
- **Min L для c≠p** = 5 (т.к. L>4)
- **Max L для c≠p** = 258 (255 + 3)
- **Min L для p** = 3 (т.к. L>2)
- **Max L для p** = 256 (255 + 1)

### **Эффективность:**
- Отлично для длинных последовательностей одинаковых символов
- Плохо для разнородных данных (может увеличивать размер)

## 🔄 **LZ78 (Вариант 2) - LZW (Lempel-Ziv-Welch)**

### **Основная идея:**
Построение словаря фраз "на лету" во время кодирования

### **Алгоритм:**
1. **Инициализация**: словарь = {все байты 0-255}
2. **w** = пустая строка
3. Для каждого символа **c**:
   - Если **w+c** есть в словаре: **w** = **w+c**
   - Иначе:
     - Вывести код для **w**
     - Добавить **w+c** в словарь
     - **w** = **c**

### **Формат вывода:**
- **Коды по 2 байта** (16 бит)
- **Максимальный размер словаря**: 4096 записей

### **Особенности:**
- Не требует передачи словаря (декодер строит его синхронно)
- Эффективен для текстов и данных с повторяющимися паттернами

## 🔍 **LZ77 (Вариант 1) - Sliding Window**

### **Основная идея:**
Поиск совпадений в "скользящем окне" предыдущих данных

### **Параметры окна:**
- **Размер окна**: 1024 байта (S:10 бит → 2¹⁰ = 1024)
- **Буфер предпросмотра**: 63 байта (L:6 бит → 2⁶-1 = 63)

### **Формат ссылки:**
- `(S:10, L:6)` - **2 байта на ссылку**
- **S** - смещение (distance) в окне
- **L** - длина совпадения

### **Алгоритм кодирования:**
1. Ищем максимальное совпадение в окне
2. Если найдено: выводим `(S, L)`
3. Если не найдено: выводим `(0, 0)` + символ

### **Преимущества:**
- Простая реализация
- Хорошо для локальных повторений

---

## 📈 **Модель Маркова и анализ эффективности**

### **Энтропия Шеннона:**
```
H = -Σ pᵢ × log₂(pᵢ)
```
где pᵢ - вероятность i-го символа

### **Теоретический предел сжатия:**
```
Размер ≥ N × H / 8 + размер_таблицы_частот
```

### **Сравнение с практикой:**
- RLE эффективен когда H → 0 (много повторений)
- LZ эффективен при наличии паттернов любой длины
- Реальное сжатие всегда хуже теоретического предела

## 💡 **Практические выводы:**

- **RLE**: идеален для растровых изображений, простых повторений
