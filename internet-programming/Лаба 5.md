## Как работает твой WebSocket-сервер — кратко и понятно

1. **HTTP сервер:**
    
    - При обычном HTTP-запросе (GET) сервер отдаёт клиенту HTML-страницу (`ws_test.html`), которая содержит код для подключения по WebSocket.
        
    - Этот сервер слушает порт (на Replit — это `process.env.PORT`).
        
2. **WebSocket сервер:**
    
    - Создаётся поверх того же HTTP-сервера, через библиотеку `ws`.
        
    - При подключении клиента (`wss.on('connection')`) создаётся уникальный ID для клиента и запускается обмен сообщениями.
        
    - Клиенты общаются с сервером по двунаправленному протоколу WebSocket, который позволяет мгновенно отправлять и получать данные без постоянных HTTP-запросов.
        
3. **Обработка сообщений:**
    
    - Клиент посылает JSON-сообщения с типами (например, `connect`, `message`, `get_history`).
        
    - Сервер обрабатывает их, например:
        
        - Назначает имя пользователя (если не указано).
            
        - Рассылает сообщения всем или приватно.
            
        - Сохраняет историю сообщений.
            
    - Сервер регулярно пингует клиентов, чтобы проверить активность и закрывает неактивные соединения.
        
4. **Преимущества WebSocket:**
    
    - Постоянное открытое соединение (не нужно открывать заново каждый раз).
        
    - Быстрая и эффективная передача данных в обе стороны.
        
    - Идеально подходит для чатов, игр, реального времени.
        
5. **Почему надо использовать `wss://` и правильный адрес:**
    
    - Replit работает через HTTPS, поэтому WebSocket тоже должен быть защищённым (`wss://`).
        
    - Клиент берёт адрес сервера из текущей страницы, чтобы соединение прошло корректно.

## Основные моменты, которые можно упомянуть:

- Сервер и клиент обмениваются данными через протокол WebSocket — это отдельный протокол поверх TCP, обеспечивающий двунаправленную связь.
    
- Сервер хранит список подключённых клиентов в `Map`, каждому даёт уникальный ID и имя.
    
- История сообщений хранится в массиве и периодически сохраняется на диск.
    
- При входе/выходе пользователей рассылаются системные сообщения всем.
    
- Используется heartbeat (ping/pong) для проверки активности клиента и отключения при бездействии.
    
- На клиенте WebSocket создаётся с правильным URL (`wss://`), чтобы избежать проблем с безопасностью и междоменными ограничениями.
