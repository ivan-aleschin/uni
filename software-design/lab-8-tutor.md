# Руководство системного программиста

## 1. Общие сведения

### 1.1 Назначение документа

Настоящее руководство системного программиста предназначено для специалистов, осуществляющих развертывание, настройку, сопровождение и техническую поддержку серверной части программной системы **AI-Travel (WayWise Backend API)**.

Документ ориентирован на системных программистов и backend-разработчиков и описывает фактическую реализацию сервера на базе **FastAPI**, включая работу с аутентификацией, генерацией текста через LLM (Ollama), базой данных и Docker-окружением.

### 1.2 Назначение системы

Сервер AI-Travel предоставляет REST API для мобильных и веб-клиентов и реализует следующие функции:

- регистрация и аутентификация пользователей;
    
- подтверждение email и номера телефона;
    
- генерация рекомендаций (страны, города, места);
    
- уточняющие запросы к AI по выбранным объектам;
    
- поиск авиабилетов и бронирование (в разработке).
    

---

## 2. Условия эксплуатации

### 2.1 Требования к аппаратному обеспечению

**Сервер:**

- CPU: 4 ядра и выше
    
- RAM: 8–16 ГБ (рекомендуется 16 ГБ)
    
- Диск: от 50 ГБ
    
- GPU: не требуется (используется Ollama), при наличии ускоряет работу моделей
    

### 2.2 Требования к программному обеспечению

- ОС: Linux (Ubuntu 20.04+) или Windows
    
- Python 3.10+
    
- Docker и Docker Compose
    
- Ollama версии **0.6.0 и выше**
    
- PostgreSQL
    

---

## 3. Архитектура системы

### 3.1 Общая схема

Система реализована в виде серверного REST API и включает следующие компоненты:

- FastAPI сервер;
    
- модуль аутентификации (JWT, OAuth2);
    
- AI-модуль генерации текста (через Ollama);
    
- СУБД PostgreSQL;
    
- Docker-контейнер для развертывания.
    

### 3.2 Backend

Backend реализован на языке **Python** с использованием **FastAPI** и выполняет:

- обработку HTTP-запросов;
    
- проверку JWT-токенов;
    
- работу с БД через SQLAlchemy;
    
- проксирование запросов к Ollama (`/api/generate`).
    

### 3.3 AI-подсистема

AI-часть использует локально развернутый сервис **Ollama** с моделями семейства Qwen:

- `qwen2.5:14b` — рекомендуемая модель;
    
- `qwen2.5:1.5b` — для систем с ограниченными ресурсами.
    

Ollama должен быть запущен отдельно командой:

```bash
ollama serve
```

## 4. Установка и развертывание

### 4.1 Подготовка AI-модели

Перед запуском сервера необходимо загрузить модель:

```bash
ollama pull qwen2.5:14b
```

или (при недостатке VRAM):

```bash
ollama pull qwen2.5:1.5b
```

### 4.2 Запуск без Docker

```bash
pip install -r requirements.txt
uvicorn main:app --host 0.0.0.0 --port 8000
```

### 4.3 Запуск в Docker (Windows)

```bash
docker-compose up --build
```

### 4.4 Запуск в Docker (Ubuntu)

```bash
docker build --tag ubuntu_waywise -f Dockerfile ./
docker run --network=host --name waywise_backend -it ubuntu_waywise
```

Внутри контейнера:

```bash
./run_backend.sh
```

Сервер доступен по адресу:

- [http://127.0.0.1:8000/](http://127.0.0.1:8000/)
    
- [http://127.0.0.1:8000/docs](http://127.0.0.1:8000/docs)

## 5. Доступ к API

### 5.1 Аутентификация

Для защищённых эндпоинтов требуется JWT-токен:

```
Authorization: Bearer <token>
```

### 5.2 Ограничения

- Ограничение запросов: **3 запроса в минуту** на пользователя
    
- Формат данных: JSON
    

---

## 6. Основные API-функции

### 6.1 Аутентификация и пользователи

- `/auth/register/` — регистрация пользователя
    
- `/auth/login/` — вход и получение токена
    
- `/auth/send_verification_code/` — отправка кода подтверждения
    
- `/auth/verify_email/` — подтверждение email
    
- `/auth/verify_number/` — подтверждение телефона
    
- `/auth/change_password/` — смена пароля
    
- `/auth/end_session/` — завершение сессии
    
- `/auth/upload_photo/` — загрузка фото профиля
    
- `/auth/get_photo/` — получение фото
    

### 6.2 Генерация рекомендаций

- `/recommend_country_json/` — рекомендации стран
    
- `/recommend_city_json/` — рекомендации городов
    
- `/recommend_places_of_interest_json/` — рекомендации мест
    

### 6.3 Уточняющие запросы

- `/clarify_country/`
    
- `/clarify_city/`
    
- `/clarify_place/`
    

### 6.4 Работа с билетами

- `/apis/search_flights/` — поиск авиабилетов
    
- `/apis/purchase_flight_ticket/` — бронирование (WIP)

## 8. Логирование и мониторинг

Система использует собственный логгер (`logger_config.py`), фиксирующий:

- регистрацию пользователей;
    
- попытки входа;
    
- ошибки верификации;
    
- сбои генерации текста.
    

---

## 9. Диагностика неисправностей

### 9.1 Ошибки аутентификации

- 401 Unauthorized — истёк или неверный токен
    
- 400 Bad Request — неверный код подтверждения
    

### 9.2 Ошибки Ollama

- **404** — неверный endpoint или порт (по умолчанию 11434)
    
- **502** — включён VPN или туннель
    
- Некорректный JSON — неподходящая модель или недостаточный лимит токенов
    

---

## 10. Структура проекта

- `auth.py` — аутентификация и верификация
    
- `generate.py` — генерация текста

- `schemas.py` — Pydantic-схемы
    
- `logger_config.py` — логирование
