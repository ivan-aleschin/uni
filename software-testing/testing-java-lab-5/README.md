# Лабораторная работа №5 - Тестирование с помощью Selenium

## Описание проекта
Проект содержит автоматизированные тесты на Selenium WebDriver для различных web-ресурсов в соответствии с требованиями лабораторной работы №5.

## Структура проекта

### Тесты (расположены в `src/test/java/org/example/`)

1. **ForumRegistrationTest.java** - Тестирование регистрации на форуме
   - Браузер: Chrome
   - Сайт: https://demoqa.com/automation-practice-form
   - Проверяет заполнение и отправку формы регистрации

2. **PhoneDirectoryTest.java** - Тестирование телефонного справочника
   - Браузер: Chrome
   - Сайт: https://demoqa.com/webtables
   - Проверяет добавление записи и поиск в справочнике

3. **VotingSystemTest.java** - Тестирование системы голосования
   - Браузер: Chrome
   - Сайт: https://demoqa.com/checkbox
   - Проверяет выбор элементов (голосование) и отображение результатов

4. **SearchTest.java** - Тестирование поиска
   - Браузер: Chrome
   - Сайт: https://demoqa.com/books
   - Проверяет функциональность поиска по книгам

5. **BaseTest.java** - Базовый класс для всех тестов
   - Содержит общую логику настройки и закрытия браузера

## Соответствие требованиям

✅ **Тестирование регистрации на форуме** - ForumRegistrationTest.java
✅ **Тестирование телефонного справочника** - PhoneDirectoryTest.java
✅ **Тестирование системы голосования** - VotingSystemTest.java
✅ **Тестирование поиска** - SearchTest.java
✅ **Использование браузера Chrome** - все тесты используют Chrome (наиболее стабильный вариант)

## Используемые технологии

- **Java 25**
- **Selenium WebDriver 4.15.0** - для автоматизации браузера
- **WebDriverManager 5.6.2** - для автоматической настройки драйверов браузеров
- **JUnit 5.10.1** - фреймворк для тестирования
- **Maven** - система сборки проекта

## Требования для запуска

1. Java JDK 17 или выше
2. Maven 3.6 или выше
3. Установленный браузер Google Chrome
4. Интернет-соединение (для загрузки WebDriver и доступа к тестовым сайтам)

## Запуск тестов

### Из командной строки:
```bash
mvn clean test
```

### Из IDE (IntelliJ IDEA, Eclipse):
1. Откройте проект
2. Дождитесь загрузки Maven зависимостей
3. Запустите тесты через контекстное меню или кнопку Run

### Запуск отдельного теста:
```bash
mvn test -Dtest=ForumRegistrationTest
mvn test -Dtest=PhoneDirectoryTest
mvn test -Dtest=VotingSystemTest
mvn test -Dtest=SearchTest
```

## Что было исправлено

### Проблемы в исходном проекте:
1. ❌ Тестовые классы находились в `src/main/java` вместо `src/test/java`
2. ❌ Отсутствовали зависимости в `pom.xml`
3. ❌ BaseTest был жестко закодирован на один браузер
4. ❌ Присутствовали лишние файлы (Main.java, Lab5TestApplicationTests.java)

### Внесенные исправления:
1. ✅ Все тестовые классы перемещены в правильную директорию `src/test/java/org/example/`
2. ✅ Добавлены все необходимые зависимости в `pom.xml`:
   - Selenium Java 4.15.0
   - WebDriverManager 5.6.2
   - JUnit 5.10.1
   - Maven Surefire Plugin 3.2.2
3. ✅ BaseTest переработан как абстрактный класс с методом `setupDriver()`
4. ✅ Каждый тест самостоятельно настраивает свой браузер через `@BeforeEach setUp()`
5. ✅ Все тесты используют Chrome (наиболее надежный вариант)
6. ✅ Добавлены подробные комментарии к каждому тесту

## Примечания

- WebDriverManager автоматически скачивает и настраивает нужные версии драйверов браузеров
- Все тесты используют явные и неявные ожидания для стабильности
- Тесты используют демонстрационный сайт demoqa.com, который специально создан для обучения автоматизации тестирования
- Chrome используется во всех тестах, так как это наиболее стабильный и распространенный браузер
