## 1. Описание используемого метода

В данной лабораторной работе реализован метод **гаммирования** с использованием **линейного конгруэнтного датчика псевдослучайных чисел (ЛКГ)**.

Метод гаммирования (шифрование наложением гаммы) заключается в том, что символы исходного открытого текста (или их числовые коды) объединяются побитово или по модулю с последовательностью псевдослучайных чисел, называемой гаммой шифра. Наложение происходит при помощи операции ИСКЛЮЧАЮЩЕЕ ИЛИ (XOR). Для дешифрования к криптограмме применяется та же самая операция с той же самой гаммой, что восстанавливает открытый текст.

Генерация псевдослучайных чисел происходит по рекуррентной формуле:
`T(i+1) = (A * T(i) + C) mod M`
где `A` и `C` — константы, `T(0)` — начальное значение (выступает в роли криптографического ключа), `M` — модуль (обычно `M = 2^b`, где `b` — длина слова).

В соответствии с теоремой Д. Кнута, линейный конгруэнтный датчик имеет максимальную длину периода (до повторения) равную `M`, если:
1. `C` и `M` взаимно просты (так как `M=2^b`, то `C` должно быть нечетным).
2. `A - 1` кратно всем простым делителям `M` (здесь это 2, значит `A` нечетное).
3. `A - 1` кратно 4, если `M` кратно 4 (то есть `A mod 4 = 1`).

## 2. Описание исходных данных

- **Исходный алфавит:** Поддерживается любой текст (русский, английский, спецсимволы), так как шифрование выполняется над числовыми кодами символов.
- **Модуль `M`:** При `b = 6` получаем `M = 64`.
- **Параметры ЛКГ (Ключ):** `A = 17`, `C = 11`, `T(0) = 7`. Условие `17 mod 4 = 1` выполняется, `11` — нечетное число, следовательно, достигается максимальный период.
- **Открытый текст:** Произвольный текст (например, `"Секретное сообщение для лабораторной работы №2! Вариант 1."`).
- **Формат данных:** Данные считываются из текстового файла `input.txt`. Результаты шифрования сохраняются в `encrypted.txt` в виде последовательности чисел, а результаты дешифрования — в `decrypted.txt`.

## 3. Алгоритм работы программы

1. **Инициализация:** Создается генератор `LinearCongruentialGenerator` и класс `GammaCipher`. В генератор передаются параметры `A`, `C`, `M` и начальный сид `T(0)`.
2. **Чтение:** Программа считывает открытый текст из файла `input.txt`. Если файл отсутствует, он создается автоматически.
3. **Шифрование:** Для каждого символа исходного текста берется его числовое представление `ord()`, генерируется очередное псевдослучайное число гаммы вызовом метода `next()` ЛКГ, после чего они складываются по модулю 2 (оператор `^`).
4. **Сохранение криптограммы:** Зашифрованные данные записываются в файл `encrypted.txt` в виде чисел через пробел.
5. **Дешифрование:** Генератор сбрасывается к начальному состоянию `T(0)`. Программа считывает числа из `encrypted.txt`, вновь генерирует ту же самую гамму и выполняет XOR, восстанавливая исходные символы с помощью `chr()`.
6. **Проверка изменения ключа (Задание 4):** Программа демонстрирует, что при изменении хотя бы одного параметра (например, `T(0) = 12`), получается совершенно иная криптограмма. Попытка расшифровать текст старой гаммой приводит к получению бессмысленного набора символов.

## 4. Текст программы

```python
"""
Лабораторная работа №2 — Шифрование данных при помощи генератора псевдослучайных чисел
Вариант 1: Линейные конгруэнтные датчики ПСЧ, b = 6
"""

import os


class LinearCongruentialGenerator:
    """
    Линейный конгруэнтный датчик ПСЧ.
    T(i+1) = (A * T(i) + C) mod M
    """

    def __init__(self, a: int, c: int, m: int, seed: int) -> None:
        self.a = a
        self.c = c
        self.m = m
        self.state = seed

    def next(self) -> int:
        self.state = (self.a * self.state + self.c) % self.m
        return self.state

    def reset(self, seed: int) -> None:
        self.state = seed


class GammaCipher:
    """
    Шифрование гаммированием с использованием ЛКГ ПСЧ.
    """

    def __init__(self, a: int, c: int, b: int, seed: int) -> None:
        self.a = a
        self.c = c
        self.b = b
        self.m = 2 ** b
        self.seed = seed
        self.prng = LinearCongruentialGenerator(self.a, self.c, self.m, self.seed)

    def encrypt(self, text: str) -> list[int]:
        """Шифрует исходный текст."""
        self.prng.reset(self.seed)
        encrypted = []
        for char in text:
            gamma = self.prng.next()
            # Наложение гаммы (сложение по модулю 2)
            encrypted.append(ord(char) ^ gamma)
        return encrypted

    def decrypt(self, encrypted_data: list[int]) -> str:
        """Дешифрует криптограмму."""
        self.prng.reset(self.seed)
        decrypted_chars = []
        for enc_char in encrypted_data:
            gamma = self.prng.next()
            decrypted_chars.append(chr(enc_char ^ gamma))
        return "".join(decrypted_chars)

    def print_params(self) -> None:
        """Печатает параметры генератора для отчёта."""
        print("  Параметры линейного конгруэнтного датчика ПСЧ:")
        print(f"    Множитель (A): {self.a}")
        print(f"    Приращение (C): {self.c}")
        print(f"    Количество разрядов (b): {self.b}")
        print(f"    Модуль (M): {self.m}")
        print(f"    Начальное значение T(0): {self.seed}")


def _banner(title: str, width: int = 60) -> str:
    line = "=" * width
    return f"\n{line}\n  {title}\n{line}"


def main() -> None:
    # Вариант 1: Линейные конгруэнтные датчики ПСЧ, b = 6
    # Максимальный период M = 2^6 = 64
    # Условия: C нечетное, A mod 4 = 1
    a = 17
    c = 11
    b = 6
    seed = 7

    cipher = GammaCipher(a, c, b, seed)

    input_file = "input.txt"
    encrypted_file = "encrypted.txt"
    decrypted_file = "decrypted.txt"

    # Создаём входной файл с примером, если он отсутствует
    if not os.path.exists(input_file):
        sample = "Секретное сообщение для лабораторной работы №2!\nВариант 1."
        with open(input_file, "w", encoding="utf-8") as f:
            f.write(sample)
        print(f"[INFO] Создан файл '{input_file}' с тестовым текстом.")

    with open(input_file, "r", encoding="utf-8") as f:
        original = f.read()

    print(_banner("ПАРАМЕТРЫ ГЕНЕРАТОРА ПСЧ"))
    cipher.print_params()

    print(_banner("ИСХОДНЫЙ ТЕКСТ"))
    print(original)

    # 1. Шифрование
    encrypted_data = cipher.encrypt(original)
    encrypted_str = " ".join(map(str, encrypted_data))
    
    with open(encrypted_file, "w", encoding="utf-8") as f:
        f.write(encrypted_str)

    print(_banner("ЗАШИФРОВАННЫЙ ТЕКСТ (КРИПТОГРАММА)"))
    print(encrypted_str)

    # 2. Дешифрование
    with open(encrypted_file, "r", encoding="utf-8") as f:
        cipher_from_file_str = f.read().strip()
        cipher_from_file = list(map(int, cipher_from_file_str.split()))

    decrypted = cipher.decrypt(cipher_from_file)
    with open(decrypted_file, "w", encoding="utf-8") as f:
        f.write(decrypted)

    print(_banner("РАСШИФРОВАННЫЙ ТЕКСТ"))
    print(decrypted)

    # 3. Верификация корректности
    print(_banner("ВЕРИФИКАЦИЯ"))
    if original == decrypted:
        print("[УСПЕХ] Текст успешно и однозначно дешифрован.")
    else:
        print("[ОШИБКА] Возникли проблемы при расшифровке!")
        
    # 4. Изменение параметра (задание 4)
    print(_banner("ИЗМЕНЕНИЕ ПАРАМЕТРА T(0) = 12"))
    changed_seed = 12
    cipher_new = GammaCipher(a, c, b, changed_seed)
    
    encrypted_data_new = cipher_new.encrypt(original)
    encrypted_str_new = " ".join(map(str, encrypted_data_new))
    
    print("  Шифрограмма с изменённым начальным значением:")
    print("  " + encrypted_str_new[:80] + "...")
    
    if encrypted_str != encrypted_str_new:
        print("\n[УСПЕХ] Шифрограммы отличаются при изменении параметра генератора.")
    else:
        print("\n[ОШИБКА] Шифрограммы совпадают!")
        
    wrong_decrypted = cipher_new.decrypt(encrypted_data)
    print("\n  Попытка расшифровать оригинальную криптограмму новым ключом:")
    print("  " + repr(wrong_decrypted[:40]) + "...")


if __name__ == "__main__":
    main()
```

## 6. Анализ результатов

В ходе выполнения программы мы убедились, что метод гаммирования с помощью ЛКГ позволяет надежно зашифровать и расшифровать исходный текст при условии полного совпадения ключей (параметров $A$, $C$, $M$ и $T(0)$) у отправляющей и принимающей стороны. Поскольку операция побитового ИСКЛЮЧАЮЩЕЕ ИЛИ (XOR) обратима, двукратное наложение одной и той же псевдослучайной последовательности (гаммы) на данные восстанавливает их исходный вид:
`Открытый_Текст ^ Гамма ^ Гамма = Открытый_Текст`

Также было наглядно продемонстрировано (п.4 задания), что шифрование крайне чувствительно к изменению ключа: изменение только начального значения генератора $T(0)$ с $7$ на $12$ полностью изменило генерируемую гамму. Это привело к совершенно иной криптограмме. Попытка дешифрования исходной криптограммы с неверным ключом выдала случайный набор нечитаемых символов.
