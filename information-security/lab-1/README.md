## 1. Описание используемого метода

В данной лабораторной работе реализован **метод моноалфавитной подстановки**. 
Это один из простейших видов криптографических преобразований, заключающийся в замене символов исходного текста на другие символы того же алфавита по определенному правилу (ключу).

Для успешной реализации этого метода размер множества символов открытого текста и шифротекста должен совпадать. В качестве ключа используется словарь подстановки английского алфавита, дополненный специальными символами. Исходная таблица по заданию содержала повторяющиеся значения (коллизии), поэтому для обеспечения **биекции** (взаимно однозначного соответствия и возможности расшифровки) некоторые повторяющиеся символы были заменены на уникальные латинские буквы.

При шифровании каждая буква исходного сообщения заменяется на соответствующую букву из ключа (подстановочного алфавита). При дешифровании выполняется обратная операция через сгенерированную реверсивную таблицу.

## 2. Описание исходных данных

- **Исходный алфавит:** Английский алфавит (`A-Z`) + специальные символы пробела и пунктуации.
- **Подстановочный алфавит (Ключ):** Словарь `SUBSTITUTION_TABLE`, в котором каждому символу исходного алфавита сопоставлен строго один символ шифроалфавита.
- **Открытый текст:** Произвольный текст на английском языке (например, `INFORMATION SECURITY IS AN IMPORTANT TASK. THIS IS LAB 1!`).
- **Формат данных:** Данные считываются из текстового файла `input.txt`. Результаты шифрования сохраняются в `encrypted.txt`, а результаты дешифрования — в `decrypted.txt`.

## 3. Алгоритм работы программы

1. **Инициализация:** Создается класс `SubstitutionCipher`. Задается прямая таблица `_encode` и автоматически генерируется обратная таблица `_decode` для расшифровки.
2. **Чтение:** Программа считывает открытый текст из файла `input.txt`. Если файл отсутствует, он создается автоматически с тестовым содержимым.
3. **Шифрование:** Исходный текст переводится в верхний регистр и посимвольно заменяется по словарю. Символы, отсутствующие в алфавите (например, цифры или знак `\n`), остаются без изменений.
4. **Сохранение криптограммы:** Зашифрованный текст записывается в файл `encrypted.txt` и выводится на экран.
5. **Дешифрование:** Криптограмма считывается из файла `encrypted.txt` и переводится обратно в исходный текст на основе обратной таблицы `_decode`.
6. **Сохранение и проверка:** Расшифрованный текст записывается в `decrypted.txt`. Программа сравнивает его с изначальным текстом и выводит сообщение об успешной верификации.

## 4. Текст программы

```python
"""
Лабораторная работа №1 — Шифрование методом подстановки
Вариант 1: Таблица 1.2, Столбец 3
Исходный алфавит: английский (A-Z + спецсимволы)
"""

import os
from typing import Final


class SubstitutionCipher:
    """
    Моноалфавитный шифр подстановки.
    """

    SUBSTITUTION_TABLE: Final[dict[str, str]] = {
        "A": "Z",
        "B": " ",
        "C": ".",
        "D": "X",
        "E": "Y",
        "F": ",",
        "G": "!",
        "H": "S",
        "I": "T",
        "J": ":",
        "K": ";",
        "L": "Q",
        "M": "R",
        "N": "?",
        "O": "-",
        "P": "N",
        "Q": "O",
        "R": "P",
        "S": "L",
        "T": "M",
        "U": "U",  # Изменено на уникальное 'U' (была коллизия: 'P' тоже давал 'N')
        "V": "V",  # Изменено на уникальное 'V' (была коллизия: 'Q' тоже давал 'O')
        "W": "W",  # Изменено на уникальное 'W' (была коллизия: 'R' тоже давал 'P')
        "X": "A",
        "Y": "B",
        "Z": "C",
        " ": "D",
        ".": "E",
        ",": "F",
        "!": "G",
        ":": "H",
        ";": "I",
        "?": "J",
        "-": "K",
    }

    def __init__(self) -> None:
        self._encode: dict[str, str] = self.SUBSTITUTION_TABLE
        # Так как теперь все значения уникальны, обратная таблица строится однозначно
        self._decode: dict[str, str] = {v: k for k, v in self._encode.items()}

    def encrypt(self, text: str) -> str:
        """
        Шифрует исходный текст.
        Символы вне алфавита пропускаются без изменений.
        """
        return "".join(self._encode.get(ch, ch) for ch in text.upper())

    def decrypt(self, text: str) -> str:
        """
        Дешифрует криптограмму.
        Символы вне алфавита пропускаются без изменений.
        """
        return "".join(self._decode.get(ch, ch) for ch in text)

    def print_table(self) -> None:
        """Печатает таблицу шифрования для отчёта."""
        print("  Таблица подстановки (биекция):")
        items = list(self._encode.items())
        mid = len(items) // 2
        for (src1, enc1), (src2, enc2) in zip(items[:mid], items[mid:]):
            s1 = repr(src1) if src1 in " ." else src1
            e1 = repr(enc1) if enc1 in " ." else enc1
            s2 = repr(src2) if src2 in " ." else src2
            e2 = repr(enc2) if enc2 in " ." else enc2
            print(f"    {s1:>4} -> {e1:<4}  |  {s2:>4} -> {e2:<4}")


def _banner(title: str, width: int = 60) -> str:
    line = "=" * width
    return f"\n{line}\n  {title}\n{line}"


def main() -> None:
    cipher = SubstitutionCipher()

    input_file = "input.txt"
    encrypted_file = "encrypted.txt"
    decrypted_file = "decrypted.txt"

    # Создаём входной файл с примером на английском, если он отсутствует
    if not os.path.exists(input_file):
        sample = "INFORMATION SECURITY IS AN IMPORTANT TASK.\nTHIS IS LAB 1!"
        with open(input_file, "w", encoding="utf-8") as f:
            f.write(sample)
        print(f"[INFO] Создан файл '{input_file}' с тестовым текстом.")

    with open(input_file, "r", encoding="utf-8") as f:
        original = f.read()

    print(_banner("ТАБЛИЦА ПОДСТАНОВКИ"))
    cipher.print_table()

    print(_banner("ИСХОДНЫЙ ТЕКСТ"))
    print(original)

    # 1. Шифрование
    encrypted = cipher.encrypt(original)
    with open(encrypted_file, "w", encoding="utf-8") as f:
        f.write(encrypted)

    print(_banner("ЗАШИФРОВАННЫЙ ТЕКСТ (КРИПТОГРАММА)"))
    print(encrypted)

    # 2. Дешифрование
    with open(encrypted_file, "r", encoding="utf-8") as f:
        cipher_from_file = f.read()

    decrypted = cipher.decrypt(cipher_from_file)
    with open(decrypted_file, "w", encoding="utf-8") as f:
        f.write(decrypted)

    print(_banner("РАСШИФРОВАННЫЙ ТЕКСТ"))
    print(decrypted)

    # 3. Верификация корректности
    print(_banner("ВЕРИФИКАЦИЯ"))
    reference = original.upper()

    if reference == decrypted:
        print("[УСПЕХ] Текст успешно и однозначно дешифрован.")
    else:
        print("[ОШИБКА] Возникли проблемы при расшифровке!")
        for i, (a, b) in enumerate(zip(reference, decrypted)):
            if a != b:
                print(f"  Позиция {i}: ожидалось {repr(a)}, получено {repr(b)}")
                break


if __name__ == "__main__":
    main()
```

## 8. Ответы на контрольные вопросы

**1. Почему метод подстановки имеет слабую надежность?**
Метод подстановки имеет слабую надежность, поскольку каждая буква исходного текста всегда заменяется на один и тот же конкретный символ шифротекста. Из-за этого зашифрованное сообщение полностью сохраняет статистические свойства открытого языка: частоты появления отдельных букв, наиболее частые биграммы (пары букв) и триграммы. Это позволяет взломать шифр без знания ключа, опираясь только на статистику.

**2. Что такое частотный анализ?**
Частотный анализ — это метод криптоанализа (взлома шифров), основанный на изучении частоты появления символов и их комбинаций в криптограмме. Аналитик сравнивает полученные частоты с известным эталонным распределением частот букв в конкретном языке (например, в английском языке буква "E" встречается чаще всего). Сопоставляя самые частые символы шифротекста с самыми частыми буквами языка, можно постепенно восстановить весь исходный текст.

**3. Что является криптографическим ключом в методе перестановки?**
В методе перестановки криптографическим ключом является само правило (алгоритм или матрица), определяющее новый порядок следования символов открытого текста. Это может быть числовой ключ, задающий порядок перестановки столбцов в таблице, или конкретный маршрут обхода символов при формировании шифрограммы.

**4. Как связаны метод подстановки и многоалфавитные шифры?**
Многоалфавитные шифры (например, шифр Виженера) являются прямым эволюционным развитием метода простой подстановки. Если в моноалфавитной подстановке для замены используется только один постоянный алфавит (словарь), то в многоалфавитных шифрах для каждой очередной буквы сообщения применяется новый подстановочный алфавит (выбираемый по определенному ключу). Это "размазывает" статистику частот символов, так как одна и та же исходная буква может быть зашифрована разными символами в зависимости от своей позиции в тексте, что делает частотный анализ гораздо более сложным.
