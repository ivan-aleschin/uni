# Лабораторная работа №5 — Создание электронной подписи в документе

Цель работы:
- Разработать процедуры выработки и проверки электронной цифровой подписи (ЭЦП) сообщений на базе асимметричного криптографического алгоритма с применением функции хеширования.
- Реализовать прототип в виде программы и оформить отчет по результатам экспериментов.

## 1. Теоретические положения (кратко)
Схема цифровой подписи обеспечивает авторство сообщения: подпись может быть выработана только при наличии секретного ключа автора и проверена с использованием открытого ключа. В варианте, используемом в этой работе, схема аналогична ГОСТ Р34.10-94 / DSA-подобной схеме:

Параметры системы (не являются секретными, могут быть общими для группы):
- `p` — простое число.
- `q` — простое число, делитель `p-1`.
- `a` — целое, `1 < a < p-1`, удовлетворяет `a^(q) (mod p) = 1` (т.е. порядок `a` делит `q`).
- `k` — случайный сеансовый ключ при генерации подписи, `1 < k < q`, уничтожается после использования.
- `x` — секретный ключ подписи, `1 < x < q`.
- `y` — открытый ключ проверки подписи, `y = a^x mod p`.

Процедура генерации подписи (обобщённо):
1. Вычисляется хэш сообщения `h = H(m)`. Если `h mod p == 0`, берём `h = 1`.
2. Случайно выбирают `k` в диапазоне `1..q-1`.
3. `r = a^k mod p`, `r1 = r mod q`. Если `r1 == 0`, берут другое `k`.
4. `s = (x * r1 + k * h) mod q`. Если `s == 0`, берут другое `k`.
5. Пара `(r1, s)` — электронно-цифровая подпись.

Проверка подписи:
1. Проверяются условия `0 < r1 < q` и `0 < s < q`.
2. Вычисляется `h = H(m)`; если `h mod p == 0`, берём `h = 1`.
3. `v = h^(q-2) mod q`  (обратный элемент `h` по модулю `q`).
4. `z1 = s * v mod q`, `z2 = (q - r1) * v mod q`.
5. `u = (a^z1 * y^z2 mod p) mod q`.
6. Подпись действительна если `u == r1`.

> Замечание: формулы приведены в нотации, близкой к ГОСТ/DSA. В реализации важно соблюдать порядок операций по модулю и корректность арифметики по модулю.

## 2. Выбранная хеш-функция (вариант 1)
По условию вариант 1 — алгоритм вычисления контрольной суммы:
- "Количество единиц в битовом представлении символов исходного текста".

Реализация `H(m)`:
- Для каждого байта (символа) текста получить его 8-битное представление.
- Посчитать суммарное количество единичных битов во всём тексте.
- Получившееся целое число взять как `h(m)`. В дальнейшем при необходимости работать с `h mod p` или `h mod q`, как указано в алгоритме.

Это простая (криптографически некрепкая) хеш-функция — вполне подходит для учебной лабораторной работы и позволяет детально отслеживать вычисления.

## 3. Алгоритм реализации (структура программы)
Программа должна:
1. Загружать текст из `input.txt`.
2. Загружать (или задавать в коде) параметры системы `p, q, a`, а также секретный ключ `x` (и формировать `y`).
   - В учебном варианте параметры можно задать константами в начале `main.py` или хранить в `keys.json`/`public_key.txt`.
3. Реализовать функцию-хеш `H(m)` по варианту 1 (число единиц в битах).
4. Реализовать генерацию подписи `sign(message, params, x)`:
   - Вычисление `h`, выбор случайного `k`, вычисление `r1` и `s`, повтор при нулевых значениях.
   - Сохранять подпись в файл `signature.txt` в читаемом виде (например `r1 s`) и/или в бинарном/JSON виде.
5. Реализовать проверку подписи `verify(message, signature, params, y)`:
   - Вычисление `h`, `v`, вычисление `u`, сравнение с `r1`.
   - Вывод результата проверки (подлинна/не подлинна).
6. Реализовать тест на изменение сообщения:
   - После получения валидной подписи изменить один символ/бит в `input.txt` (или в памяти) и показать, что проверка теперь не проходит.
7. Вывести пошаговые промежуточные значения (для отчёта): `h`, `k`, `r`, `r1`, `s`, `v`, `z1`, `z2`, `u`.

Файлы (рекомендуемые):
- `main.py` — основная программа (генерация и проверка).
- `input.txt` — исходное сообщение (текст).
- `signature.txt` — подпись (например содержимое: `r1=<число> s=<число>`).
- `public_key.txt` — открытые параметры (`p,q,a,y`) (по желанию).
- `private_key.txt` — (или хранить в коде) секретный ключ `x` (для учебной работы может быть в коде).

Формат вывода:
- Читаемые значения в консоль с разделителями — используйте баннеры, аналогично оформлению в предыдущих лабораторных (`===` линии).
- Также сохранить результаты (подпись, расшифровки) в отдельные файлы.

## 4. Как запускать (пример)
1. Перейдите в каталог лабораторной работы:
   - `cd uni/information-security/lab-5`
2. Запустите программу:
   - `python3 main.py`
3. Ожидаемый поток действий в программе:
   - Программа выводит параметры системы (`p, q, a, y`), вычисляет и печатает хеш `h`.
   - Выполняет генерацию подписи, печатает `k, r, r1, s`.
   - Сохраняет подпись в `signature.txt`.
   - Выполняет проверку подписи и выводит результат.
   - Затем демонстрация: вносится/моделируется изменение сообщения, повторная проверка — подпись должна быть отвергнута.

> Пример (примерные строки вывода, не как код- блок):
- `p = 46337`
- `q = 233`
- `a = 5`
- `y = 12345`
- `H(m) = 1024`
- `k = 87 -> r = 34567 -> r1 = 123`
- `s = 201`
- `Signature: r1=123 s=201`
- `Verification: VALID`
- После изменения сообщения:
- `Verification: INVALID`

## 5. Что проверить для корректности (и на что обратить внимание)
- Проверить, что при генерации подписи `0 < r1 < q` и `0 < s < q`. При нарушениях нужно генерировать новый `k`.
- Проверить корректность вычисления обратного по модулю при `v = h^(q-2) mod q` (или вычислять модинверс напрямую при необходимости).
- Убедиться, что при изменении сообщения проверка не проходит.
- Для небольших параметров (учебных p, q) — можно вывести отдельные шаги, чтобы вручную сверить результаты.
